# 扬说财经 · 每日改进清单

## 每日结束前（必须执行）

```bash
bash scripts/daily-upgrade.sh
```

该脚本自动执行以下 10 个步骤。以下是每个步骤的检查要点：

### Step 1: 数据索引更新
- [ ] `scripts/update-index.py` 运行成功
- [ ] `data/articles.json` 和 `data/articles.js` 均已更新

### Step 2: 部署前检查
- [ ] `scripts/pre-deploy-check.sh` 通过（或已确认警告可忽略）

### Step 3: 质量检查
- [ ] `scripts/site-health.py` 通过（非阻塞）

### Step 4: 文章完整性
- [ ] 当天的早报 `article.html` 存在
- [ ] 当天的晚报 `article.html` 存在
- [ ] 所有漫画 SVG 文件存在
- [ ] 音频文件存在（或标注为待生成）

### Step 5: 首页预渲染
- [ ] `index.html` 中 `data-date` 属性指向当天
- [ ] 早报链接包含正确的文章标题
- [ ] 晚报链接包含正确的文章标题

### Step 6: 数据一致性
- [ ] `articles.json` 中文章数量 > 0
- [ ] 最新日期正确

### Step 7: 部署
- [ ] 部署成功（HTTP 200）

### Step 8: 部署后验证
- [ ] 首页 200 ✅
- [ ] 数据文件 200 ✅
- [ ] 早报文章页 200 ✅
- [ ] 晚报文章页 200 ✅
- [ ] 首页源代码包含预渲染文章链接 ✅

### Step 9: 版本标记
- [ ] Git tag 已创建并推送

### Step 10: 问题记录
- [ ] 今天遇到的问题已记录到日志

---

## 每日生产前（负责人检查）

- [ ] 网站可以正常访问
- [ ] 前一天的部署验证通过
- [ ] 前一天的问题已处理
- [ ] 今天的财经热点已整理

---

## 每周复盘（周日）

1. **运行周报告：** 统计本周部署次数、回滚次数、故障次数
2. **问题分类：** 将本周问题归类到"架构/流程/执行"三层
3. **改进追踪：** 检查上周改进项的完成情况
4. **知识库更新：** 将新学到的经验教训写入 `docs/post-mortem-*.md`
5. **下周计划：** 确定 Top 3 改进方向
