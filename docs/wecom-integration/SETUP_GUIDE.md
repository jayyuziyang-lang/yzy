## 你的下一步操作

### Step 1: 安装 OpenClaw Daemon（管理员权限）

用管理员身份打开终端，运行：
```bash
openclaw daemon install
```
这样电脑开机后 Gateway 会自动启动，无需手动操作。

### Step 2: 在企业微信中创建群组

| 群名 | 用途 | 成员 |
|------|------|------|
| **扬说财经·总指挥部** | 你与机器人的主沟通群 | 你 + 机器人 |
| **扬说财经·生产前线** | Agent生产日报讨论 | 你 + 机器人 |
| **扬说财经·质量审核** | 内容质检和审批 | 你 + 机器人 |

创建步骤：
1. 打开企业微信 → 点击"+" → 「创建群聊」
2. 搜索并添加机器人（显示为"扬说财经"或自定义名称）
3. 发送任意消息测试机器人是否回复

### Step 3: 测试机器人

在任意群中发送消息给机器人，例如：
```
@机器人 今天有什么新闻？
```

如果机器人回复了，说明整个链路通了。

### Step 4: 验证定时任务

等待明天 7:30 AM 看看是否自动触发早报生产。也可以手动测试：
```bash
openclaw cron run morning-report
```

如果报 `scope upgrade` 错误，请先用管理员终端运行一次：
```bash
openclaw daemon install && openclaw gateway restart
```

## 使用方式

### 半自动模式（当前）

定时任务会自动生产内容，但**不会自动发布**。你需要：

1. 在群中问机器人："今天早报做好了吗？"
2. 机器人会汇报生产状态
3. 你审核后说："可以发布了"
4. 机器人执行 `update-index.py` 并推送 GitHub

### 场景示例

**场景1：催更早报**
> 你：今天的早报怎么样了？
> 机器人：早报已完成！A股数据已采集，6个故事已撰写，SVG和音频已就绪。
> 我帮你预览文章内容...
> 你：发布吧
> 机器人：已执行 update-index.py，推送 GitHub Pages。

**场景2：查询数据**
> 你：昨天美股收盘数据
> 机器人：道琼斯... 标普500... 纳斯达克...

**场景3：要求修改**
> 你：晚报第二个故事的数据有误，改成昨天收盘数据
> 机器人：已修正，重新核实了数据来源。请确认新版。

## 故障排查

| 问题 | 解决方法 |
|------|---------|
| 机器人不回复 | 检查 Gateway 是否运行: `curl http://127.0.0.1:18789/health` |
| WebSocket断开 | 自动重连（最多10次），等待30秒即可 |
| 定时任务没触发 | 执行 `openclaw cron list` 查看任务状态 |
| "scope upgrade"错误 | 用管理员终端运行 `openclaw daemon install` |
| 数据不对 | 机器人会自动标注数据来源，你可要求交叉验证 |
