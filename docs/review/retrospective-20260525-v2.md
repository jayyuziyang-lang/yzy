## 一、问题清单与根因

### P0: 5.25早报"没有更新到网站"（用户反馈3次）

**直接原因：** 
- 首次推送（commit `5a42cdb`）后 GitHub Pages 需要 1-2 分钟部署
- 用户看到的是 CDN 缓存中的旧页面
- 后续多次推送（VIX修复、索引更新、样式调整等）导致部署不断重启，延长了生效时间

**根本原因：**
- 没有在推送后立即验证部署结果
- 推送次数过多（4次），每次都会重启 GitHub Pages 构建
- 没有告知用户"部署需要等待1-2分钟"

**修复：**
- ✅ 最终部署验证通过：index.html data-date="2026-05-25" + articles.js含早报
- ✅ 文章可访问：HTTP 200 (24KB)

### P1: 深度分析专题发布日期从5.23被改为5.25

**直接原因：** 运行 `update-index.py` 时，该脚本使用 `os.path.getmtime(special_dir)` 确定专题日期。git checkout / 文件操作改变了目录 mtime，导致日期被重置。

**根本原因：**
- 没有先阅读 `update-index.py` 的代码就运行了它（违反"先读再改"原则）
- 脚本本身设计缺陷：依赖 mtime 而非固定元数据

**修复：**
- ✅ 新增 `.date` 文件存储专题固定日期
- ✅ 修改 `update-index.py` 优先读取 `.date` 文件

### P2: 不必要地修改了 index.html

**直接原因：** 认为"预渲染内容没有更新24日/25日文章"是问题，手动修改了 index.html 的 pre-rendered HTML。

**根本原因：**
- 没有理解 index.html 的设计——预渲染内容只是静态 fallback，JavaScript 在加载时根据 `ARTICLES_BY_DATE` 动态渲染当日内容
- 预渲染内容只要链接有效即可，data-date 标记与 JS 渲染无关

**教训：**
- index.html 的预渲染内容不需要人工更新 —— `renderToday()` 函数会自动处理
- 每次修改 index.html 都有引入排版/JS 错误的风险
- 不应移除 `fetchVix()` 等看起来"未使用"的函数（即使 VIX 改用静态数据，移除也非必要）

### P3: VIX 数据抓取反复失败，多轮修复

**直接原因：** Yahoo Finance API 从该环境返回 HTTP 403（IP 被限流）。

**修复：**
- 安装 yfinance 库（同样被限流）
- 增加备用数据机制（使用上一次有效数据）
- 备用数据值（VIX=16.76）已在 WebSearch 交叉验证中确认

### P4: Python 图表数据与文章脱节（5.25复盘已记录）

**直接原因：** `generate-charts.py` 使用固定硬编码数据，与当日叙事无关。

**修复：**
- ✅ 新建 `charts_today.py`，按日编写与叙事对齐的图表
- ✅ WORKFLOW.md v5.1 增加图表相关性检查

## 三、整改措施（强制）

### 3.1 首页修改禁令

**index.html 在以下情况下可以修改：**
- ✅ 新增 JS 功能（如新的动态模块）
- ✅ 修复 JS 错误
- ❌ 不得手动更新预渲染内容（JS 自动处理）
- ❌ 不得移除看似"未使用"的函数（除非100%确认）
- ❌ 不得为了"看起来更新"而修改

**预渲染内容的作用：** 仅是静态 fallback。用户浏览器加载页面后，`renderToday()` 会读取 `ARTICLES_BY_DATE` 动态渲染当日内容。

### 3.2 Python 脚本修改规范

- **运行已有脚本前：先读代码**（理解输入输出，不假设）
- **修改脚本前：先测当前行为**（备份数据文件）
- **update-index.py 特殊规则：** `special/*/.date` 文件是日期唯一来源，不要依赖 mtime

### 3.3 部署规范

- **推送后等待 2 分钟再验证**
- **验证清单（强制逐一检查）：**
  1. `curl index.html | grep data-date` → 确认 data-date 为当日
  2. `curl articles.js | grep latest` → 确认 latest_date 为当日
  3. `curl article.html | wc -c` → 确认 >10KB
  4. `curl vix-data.js | grep current` → 确认 VIX 有数据
- **告知用户：** "已推送，GitHub Pages 约 1-2 分钟后生效"

### 3.4 质量门禁强化

当前质量门禁检查清单还不够。新增检查项：

```
□ update-index.py 不可随意运行（先读代码）
□ 未修改 index.html 的非必要部分
□ 推送后等待 2 分钟部署，再验证
□ 未移除任何函数/代码行（除非明确知道后果）
```
