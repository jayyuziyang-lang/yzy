# Dalio 简笔科普漫画 · 生产执行书 v1.0

> 对标: Ray Dalio《经济机器是如何运行的》(How the Economic Machine Works)
> 核心理念: 简笔小人 + 具体物件 + 白底 + 清晰标注 → 一眼就懂
> 工具: Python SVG 代码生成（非 SD/非手绘/非 AI 生成）
> 版本: v1.0 | 生效: 2026-06-17

---

## 一、为什么是这种风格

Ray Dalio 用 30 分钟动画把「信贷→短期债务周期→长期债务周期→去杠杆」讲得比任何经济学教材都清楚。他的视觉语言有三个核心原则：

| 原则 | Dalio 做法 | 我们怎么做 |
|------|-----------|-----------|
| **极简构图** | 白纸背景，每帧 3-5 个元素 | SVG 白底 `#FAFBFC`，禁止渐变/阴影/纹理 |
| **人物即符号** | 简笔画小人代表买方/卖方/银行/央行 | `stick_figure()` 函数，圆头+线段四肢 |
| **物件即概念** | 房子=资产、齿轮=机器、线=生产力 | 工厂/温度计/天平/骰子——每个物件对应一个经济概念 |

**和原有漫画的区别：**
- 原有：300×220，几何图形，无人物，< 6KB
- Dalio：768×512，简笔画人物+具体物件+中文标注，2-5KB
- 不变：矢量 SVG，白底，早报蓝 `#1A56DB`

## 二、漫画设计执行流程（每次生产必走）

### Step 0: 读取当日新闻 → 提取视觉要素

```
输入: article.html 的 6 个故事
输出: 每故事的「三个一」
  → 一个核心矛盾（如：生产强 vs 消费弱）
  → 一个具体物件（如：工厂、温度计、天平）
  → 一个动作/状态（如：冒烟 vs 关门、热 vs 冷、倾斜）
```

### Step 1: 隐喻设计（最重要的一步）

**设计原则：**
1. **物件必须是日常可见的** — 工厂、温度计、天平、骰子、火箭、皮筋。抽象概念（"传导链""结构性矛盾"）用具体物件表达
2. **对比即叙事** — 左/右、上/下、热/冷、重/轻。Dalio 的核心技巧是「并置对比」
3. **标签要说人话** — "制造业PMI 52↑"而不是"生产端景气度回升"
4. **一问检验** — 「我妈能看懂这张图吗？」如果不能，简化隐喻

**隐喻设计模板（每张漫画填写）：**

```
□ 核心矛盾: ________ / ________（两个对立面）
□ 代表物件: ________（左/上/热/重）vs ________（右/下/冷/轻）
□ 动作状态: ________ vs ________
□ 关键数字: ________（≤ 2 个）
□ 一句话: ________（14字以内，放在图下方）
```

**示例（6.16 早报）：**

| 故事 | 核心矛盾 | 物件(左/右) | 动作 | 关键数字 | 一句话 |
|------|---------|------------|------|---------|--------|
| A股 | 生产热 / 消费冷 | 工厂烟囱 / 商场卷帘门 | 冒烟 vs 关门 | PMI 52↑, 社零3%↓ | 结构性矛盾：经济冷热不均 |
| 美股 | 狂欢 / 风险 | 铜牛+小人 / FOMC悬崖 | 跳舞 vs 逼近 | — | 最大的风险是「觉得自己没有风险」 |
| 科技 | 股价 / 基本面 | 火箭 / 地面锚桩 | 冲天 vs 钉住 | 市值$2.5万亿, +42% | 皮筋拉得太长，一定会弹回来 |
| 国际 | 日本热 / 中国冷 | 红温度计 / 蓝温度计 | 冲顶 vs 低位 | 日经70000, BOJ 1% | 亚洲经济温差：一个发烫一个发冷 |
| 宏观 | 未知 / 等待 | 悬空骰子 / 会议桌 | 未落地 vs 仰望 | — | FOMC点阵图：结果落地才算数 |
| 商品 | 利率 / 避险 | \$砝码 / 小火苗 | 重 vs 轻 | \$4300 | 黄金不是避险，是利率预期 |

### Step 2: 构图设计

**固定画布：** 768×512，白底 `#FAFBFC`

**布局规范：**
- 标题：顶部居中，y=36，字号20，早报蓝 `#1A56DB`，加粗
- 主体：y=60~400，左右分栏或居中
- 一句话总结：底部居中，y=465，字号14，深灰 `#475569`

**物件尺寸约束：**
- 简笔画小人：头 r=8-10，身高 30-45px
- 建筑类物件：宽 120-200，高 100-180
- 标注文字：12-15px，不重叠，最多 2 行/物件

### Step 3: SVG 代码生成

**使用 `gen-dalio-comics.py` 模板（见附录A）。**

核心函数：
- `stick_figure(x, y, h, color, action)` — 简笔画小人
- `label(x, y, text, size, color)` — 文字标注
- `arrow(x1, y1, x2, y2, color)` — 箭头
- `dashed_line(x1, y1, x2, y2, color)` — 虚线

**代码规范：**
- 不写重复的 SVG 字符串，用函数封装
- 一个故事一个 Python 函数（`comic_01()` ~ `comic_06()`）
- 颜色统一用变量/常量（`BLUE = "#1A56DB"`, `RED = "#DC2626"`, `GRAY = "#475569"`）

### Step 4: 质量审查（6 项自检）

```
□ [XML语法]       Python xml.etree.ElementTree.parse() 可解析
□ [font-family]   <svg> 根标签有 font-family
□ [尺寸]          viewBox="0 0 768 512"
□ [无杂物]        无渐变、无投影、无filter、无动画
□ [标注清晰]      每个数字/概念有文字标注，读者不需要猜
□ [颜色合规]      红涨(#DC2626)绿跌(#16A34A)，早报蓝(#1A56DB)
```

## 四、生产集成（在 WORKFLOW.md 中的定位）

Dalio SVG 漫画替代原有 Phase 3 的 SVG 几何漫画。生产流程：

```
Phase 0: 环境验证 → 同 WORKFLOW.md
Phase 1: 新闻采集 → 同 WORKFLOW.md
Phase 2: article.html 撰写 → 同 WORKFLOW.md
Phase 3: Dalio 漫画生产（本文件）
  → Step 0: 读取 article.html，提取6故事
  → Step 1: 每故事填写「三个一」隐喻设计模板
  → Step 2: 构图设计（768×512）
  → Step 3: 运行 gen-dalio-comics.py 生成 SVG
  → Step 4: 6项自检
Phase 4: 音频制作 → 同 WORKFLOW.md
```

**和原 WORKFLOW.md Phase 3 的冲突项（以本文为准）：**
- 尺寸：768×512（非 300×220）
- 人物：允许简笔画小人（圆头+线段四肢，非写实角色）
- 文件大小：2-5KB（非 < 6KB，实际通常更小）
- 背景：`#FAFBFC`（非 `#FAFAFA`，更接近白板）

## 附录B：质量反例（避免）

| 反例 | 问题 | 改正 |
|------|------|------|
| 画面太满，8个物件挤在一起 | 违反「极简」原则 | 减到3-5个核心元素 |
| 抽象隐喻（"裂谷""风暴云"）无标注 | 读者不懂什么意思 | 每个物件旁加中文标签 |
| 水彩/渐变/阴影/纹理 | SVG不该有这些 | 纯色块 + 实线描边 |
| 颜色用错（绿色=上涨） | 违反红涨绿跌铁律 | `RED="#DC2626"` 涨, `GREEN="#16A34A"` 跌 |
| 小人不标身份 | 不知道代表谁 | 旁标注"交易员""央行""买方" |
