# 扬说财经 · 生产工作流程 v4.0

> 版本: v4.0 | 更新: 2026-05-22
> 核心变更: 6故事分类法 + 用户确认门禁 + Codex协作升级 + 质量检查升级

---

## 核心原则

- **6故事固定分类** — 每天内容按01-06编号展开，顺序固定
- **数据+叙事+洞察** — 每条故事必须覆盖三维度，缺一不可
- **专职核查** — 事实核查员独立运作，可阻塞流水线
- **独立质检** — 质检审查官不参与生产，专门找问题
- **先假设后验证** — 每个Agent启动输出假设，完成输出证据
- **早晚分离** — 早报和晚报是两次独立的生产调用
- **日报读晚报** — 生产早报前必须检查前日晚报内容，确保Layer 2不重复
- **用户确认门禁** — 重大变更必须先展示预览，用户确认后执行
- **Codex协作** — SVG生成委托Codex，Claude Code负责审查
- **每日复盘** — 问题不过夜，每条问题落实到下轮生产

---

## 五层质量控制体系（v4.0升级）

```
Layer 1: Agent 1（新闻采集员）→ 数据源头质量
Layer 2: Agent 2（事实核查员）→ 数据来源追溯 ← 独立门禁，可阻塞
Layer 3: Agent 3/4/5（生产者）→ 内容 + SVG + 音频自检
Layer 4: Agent 7（质检审查官）→ 客观证据收集 ← 独立门禁，可驳回
Layer 5: Claude Code（主编统筹）+ 用户确认 → 终审门禁
```

**v4.0新增质检维度：**
- 6故事结构完整性检查
- 每故事三维度覆盖检查（数据+叙事+洞察）
- 空洞套话扫描
- 素描风格合规检查
- 口播稿与文章一致性检查

## 早报生产流程（11阶段，v4.0升级版）

### Phase 0: 环境验证（主编）
```
Think 假设:
1. [工具链可用性假设]
2. [数据源可用性假设]
3. [前日晚报已检查] — 确保Layer 2不重复

验证:
→ python --version
→ git status (仓库干净)
→ 生产目录就绪
→ edge-tts 可用
```

### Phase 1: 新闻采集（Agent 1 / Claude Code）
按6故事分类搜索隔夜全球财经新闻：
- 01 A股 → 昨日行情数据
- 02 美股 → 隔夜收盘数据
- 03 科技 → 产业动态
- 04 国内政策 → 政策突发
- 05 宏观 → 央行/债市/美联储
- 06 商品 → 黄金/能源

**输出:** `docs/review/news-morning-{YYYY-MM-DD}.md`
**完成后输出验证证据块。**

### Phase 2: 事实核查（Agent 2）
逐条验证新闻的数据点来源，重点核查：
- 所有百分比数字有来源
- 所有价格数字标注市场和时间
- 无"历史第X次"表述
- 关键数据至少有2个独立来源

**发现编造数据 → 阻塞流水线。**

### Phase 3: 结构设计（Claude Code + 用户确认 ← 新增）
```
【新增步骤】设计文章结构草稿，展示给用户确认:
1. 6故事标题+各故事的数据点
2. 6个SVG的视觉隐喻设计
3. 金句收尾

等待用户反馈 → 确认后进入Phase 4
```

**用户连续2次指出同一方向错误 → 停下来追问，不再猜测。**

### Phase 4: 文章撰写（Agent 3 / Claude Code）
写入 article.html，遵循6故事结构。
- Layer 1: 热点速览（panel-000）
- Layer 2: 6个编号故事（panel-001~006，每故事配SVG）
- Layer 3: 数据表 + 图表

**强制要求：** 每条故事必须包含数据+叙事+洞察三维度。

### Phase 5: SVG设计（Claude Code 设计 → Codex CLI 生成 ← 新增协作）
1. **Claude Code 设计**：输出每个SVG的视觉隐喻描述、布局草图、色彩方案
2. **Codex CLI 生成**：批量生成 panel-000 ~ panel-006 的SVG代码
3. **Claude Code 审查**：逐条检查素描风格合规性，不符合的标记退回

**审查清单：**
- [ ] 线稿为主（非信息图）
- [ ] 色彩不超过3种
- [ ] 视觉隐喻清晰
- [ ] 文件 < 30KB
- [ ] 无渐变/阴影

### Phase 6: 音频制作（Agent 5）
1. 写口播稿 script.txt：遵循6故事结构，口语化表达
2. 主编审核
3. 合成 audio.mp3

**v4.0新要求：** 口播稿必须与文章内容对齐（故事顺序、关键数据一致）

### Phase 7: 质量审查（Agent 7 / Claude Code）
独立执行质检，收集客观证据。

**检查清单（v4.0升级版）：**
- [ ] 6故事结构完整（01-06）
- [ ] 每故事含数据+叙事+洞察
- [ ] 无空洞套话（"值得关注""后市可期"等）
- [ ] 无"历史第X次"类表述
- [ ] SVG为素描风格
- [ ] 文件大小合规
- [ ] 合规声明存在
- [ ] 口播稿与文章一致

**全部通过才放行，有问题直接驳回。**

### Phase 8: 用户预览（新增门禁）
```
展示预览内容给用户:
1. 最终文章HTML预览
2. 新版SVG各面板效果
3. 口播稿

等待用户确认 → 确认后进入Phase 9
用户不满意 → 回到对应Phase修改 → 重新审查 → 重新预览
```

### Phase 9: 部署（Agent 8）
```
git add <修改文件>
git commit -m "feat: 扬说财经早报 {日期} — {改动要点}"
git push origin main
验证URL可访问
```

### Phase 10: 复盘（Agent 9）
当日总结 → 改进项记录。
输出: `docs/review/daily-{YYYY-MM-DD}.md`

## 晚报生产流程

复用早报流程，但内容导向不同：

| 维度 | 早报 | 晚报 |
|------|------|------|
| 核心 | 隔夜全球新鲜事件 | A股当日深度复盘 |
| 01 A股 | 简要数据回顾（1段） | 完整深度分析 |
| 02-06 | 全球焦点展开 | 延续深度分析+明日展望 |
| 用户期待 | 晨间快速了解发生了什么 | 晚间理解为什么发生 |

- [ ] Think假设块是否输出（每个Agent启动时）

- [ ] 数据来源是否全部可追溯
- [ ] 6故事结构是否完整
- [ ] 每故事是否含数据+叙事+洞察
- [ ] 质检是否通过（Agent 7 报告）
- [ ] 用户确认门禁是否执行
- [ ] SVG是否符合素描风格标准
- [ ] 口播稿与文章是否一致
- [ ] 改进项是否记录并落实
