## 核心原则

- **6故事固定分类** — 每天内容按01-06编号展开，顺序固定（红绿紫蓝金银橙）
- **数据+叙事+洞察** — 每条故事必须覆盖三维度，缺一不可
- **质量门不可绕过** — 四条质量门禁（事实核查→风格→质量→数据准确性），Claude Code终审
- **漫画按需配图** — 不固定数量，聚焦关键热点和复杂逻辑，不为凑数而画
- **Python工具链不可跳过** — `generate-charts.py` 每天至少运行一次
- **铁律遵守** — 详见 `TEAM.md` 7条铁律，每条都是付过学费的教训
- **早晚分离** — 早报和晚报是两次独立的生产调用
- **日报读晚报** — 生产早报前必须检查前日晚报内容，确保Layer 2不重复
- **用户确认门禁** — 重大变更（风格/结构/新增元素）必须先展示预览，用户确认后执行
- **用户连续2次指出同一方向错误 → 停下来追问，不再猜测**

### Phase 1: 新闻采集 + 事实核查（Codex/DeepSeek主力，Claude Code复核）

按6故事分类搜索全球财经新闻，逐条标注来源：

| 编号 | 分类 | 搜索重点 |
|------|------|---------|
| 01 | A股 | 昨日行情数据、板块资金流向 |
| 02 | 美股 | 隔夜收盘数据、重要公司财报 |
| 03 | 科技/AI | 产业动态、巨头新闻 |
| 04 | 前沿/国际 | 地缘政治、国际政策 |
| 05 | 宏观 | 央行/美联储/债市/经济数据 |
| 06 | 商品/黄金 | 原油/黄金/大宗商品 |

**事实核查强制标准：**
- 每个百分比数字 → 来源URL
- 每个价格数字 → 具体市场+时间
- 每个历史对比 → 精确参照年份
- 英文公司名 → 首次出现中英对照
- **禁止使用模型内部知识替代实时搜索**

**发现编造数据 → 整篇退回，根因分析。**

### Phase 2: 文章撰写（Codex初稿 / Claude Code润色）

读取风格样本 → 写入 article.html：

- **风格样本：** `special/央行法修订/article.html`（配色/字体/间距/色值参照）
- **早报开头强制：** 必须包含"今日热点速览"（4条编号要点，覆盖当日最核心事件）
- **6故事结构完整**，每故事独立成篇，不合并
- **数据+叙事+洞察** 三维度覆盖
- **禁止套话：** "值得关注""后市可期""需警惕"
- **禁止编造历史对比：** "历史第X次"等必须有精确来源
- **公司名：** 首次出现中英对照，后续只用中文

### Phase 3: 漫画生产（Claude Code设计+生成，质量门控）

**⚠️ 启动前强制操作：必须先读取风格参照文件，才能开始生产漫画。禁止凭记忆生产。**

**风格基准（双重参考）：**

| 场景 | 参照文件 | 核心特征 |
|------|---------|---------|
| 日常版 | `2026-05-22/finance_daily_20260522.html` | 300×220，线稿为主，简洁数据展现 |
| **专题/深度版** | `special/央行法修订/comic/panel-*.svg` | **300×220，建筑学隐喻，金色+深蓝配色体系，叙事驱动** |

**强制风格参数（两组统一）：**

| 维度 | 标准 |
|------|------|
| 背景色 | `#FAFAFA`（暖白）或 `#faf8f5`，不渐变，无纹理 |
| 线条色 | `#1a1a1a`（近黑），2-3px线宽，无阴影 |
| 金色强调 | `#c9a96e` / `#D4A017` |
| 上涨/利好 | `#2e7d32`（绿） |
| 下跌/风险 | `#c62828`（红） |
| viewBox | `300 220`，固定尺寸 |
| 字体 | `PingFang SC, Microsoft YaHei, sans-serif` |
| 人物/角色 | **不加入** |
| 装饰效果 | **无**：无渐变、无投影、无filter、无发光 |
| 文件大小 | **< 5KB**，保持简洁 |

**漫画数量规则：**
- 不固定数量，**不需要每故事配一张漫画**
- 聚焦：关键热点、复杂逻辑、难以用文字说清的概念
- 专题版：3-4格，每格独立叙事，配金色+深蓝色系

**文字排版规则：**
- 文字必须放在空白区域，不得与图形重叠
- 重要数字用大号粗体（11-16px），说明用小号（7-8px）
- 上下左右保留充足边距

**漫画生产确认清单（硬性要求，不逐一过不准推送）：**
```
□ 已读取5.22晚报漫画作为风格参照（panel-001~006）
□ 背景色 = #FAFAFA（浅色，无渐变/无深色/无纹理）
□ 以几何图形/线条/图标为主，不以文字堆砌为主
□ 每张漫画关键数字 ≤ 4个
□ 标注文字 ≤ 2行（每行 ≤ 20字）
□ 文件大小 < 6KB
□ SVG viewBox = 300 220
□ 无人物角色（阿扬不出现在非专题漫画中）
□ 每张漫画对应正确的故事（panel-001 = A股，非其他内容）
```

### Phase 4: 音频制作

1. 写口播稿 script.txt（阿扬人格化，第一人称）
2. 脚本字数匹配版次标准
3. 合成MP3：`python -m edge_tts --voice zh-CN-YunyangNeural --rate=-5% -f script.txt --write-media audio.mp3`

**脚本规范：**
- 纯中文文本，无英文字母/URL/HTML标签
- 金融专有名词（NVIDIA/OpenAI等）允许保留，但尽量用中文
- **口语化、人格化表达** — 多用"有意思的是""说明什么？""这个要心里有数""大家要注意"等互动语言
- **有故事感** — 不是念新闻，是阿扬在"讲给你听"。要有情绪起伏和叙事节奏
- **冷启动不要直接跳到新闻** — 先打招呼再进入正题

**开口/结尾固定模板（必须遵守，不可简化）：**

| 版次 | 开口 | 结尾 |
|------|------|------|
| 早报 | "朋友们早上好，我是阿扬。" | "今天的故事讲完了，晚上还有晚报，我会带大家深度复盘今天的行情。我是阿扬，我们晚上见。" |
| 晚报 | "各位朋友晚上好，我是阿扬，欢迎来到今天的扬说财经晚间版。" | "我是阿扬，我们明天早上见。" |
| 专题 | "朋友们好，我是阿扬。" | "我是阿扬，我们下期见。" |

**时长标准：**

| 版次 | 字数 | 预计时长 |
|------|------|---------|
| 早报 | 500-800字 | 2-3分钟 |
| 晚报 | 1500-2500字 | 6-8分钟 |
| 专题 | 1600-2700字 | 6-10分钟 |

**铁律：** 标注不撒谎。脚本字数→预计时长→实际时长，三者必须一致。

### Phase 5: 数据可视化生产与质量门禁（v2.0升级）

**核心工具链：**
| 组件 | 位置 | 用途 |
|------|------|------|
| 通用图表生成 | `scripts/generate-charts.py` | 10张固定模板图表（指数/板块/商品等） |
| 高级模板库 | `scripts/chart_templates.py` | 6种进阶图表类型（棒棒糖/瀑布/斜率等） |
| 品牌风格 | `stylelib/yangshuo.mplstyle` | 全局 rcParams 统一配置 |
| 品牌配色 | `stylelib/yangshuo_palette.py` | 红涨绿跌配色工具函数 |

**生产流程（Phase 0 → Phase 5 连贯执行）：**

```
Phase 0: 运行 python scripts/generate-charts.py
         ↓ 自动执行 "图表质量门禁 v1.0"
         ↓ 输出 30 项检查结果（文件存在 + 大小 + SVG格式）
Phase 5: 将生成图表嵌入文章HTML
         ↓ 逐图检查：颜色正确 + 标签不重叠 + 与叙事相关
```

**数据可视化质量门禁（Layer 2.5 — 新增，位于 Layer 2 风格一致性与 Layer 3 质量检查之间）：**

每张嵌入文章的图表，逐项检查：

```
□ 颜色正确性
   ├── 红涨绿跌（A股/全球指数）：正→#DC2626，负→#16A34A
   ├── 红流入绿流出（资金流向）：正→#DC2626，负→#16A34A
   └── 半导体/科技：紫色系（#7C3AED），金价：金色系（#D4A017）

□ 标签布局
   ├── 无文字堆叠/重叠（手动检查渲染结果）
   ├── 正值标签在柱右侧，负值标签在柱左侧
   └── 标注字体 ≥ 8px，保持可读

□ 数据准确性
   ├── 图表数据与文章正文一致（如指数涨跌幅数字核对）
   └── 无编造历史数据（"历史第X次"等必须有来源）

□ 视觉品质
   ├── 品牌背景色 #F8FAFC（非默认白色）
   ├── 白边框（edgecolor=WHITE, linewidth=1.5）
   ├── 网格线淡色（alpha=0.15）
   └── 删除上边框和右边框（spines top/right False）

□ 图表嵌入
   ├── 图表与对应板块叙事直接相关（非通用数据硬塞）
   ├── img 标签含 loading="lazy"
   └── caption 说明文字包含关键数据解读
```

**如果 embed 阶段发现问题 → 返回修改，不得跳过。**

### Phase 6: 质量终审（Claude Code 不可替代）

四条质量门禁逐条检查：

**Layer 1: 事实核查** — 每个数据点有来源，无来源=不发表
**Layer 2: 风格一致性** — 文章匹配央行法修订样本，漫画匹配风格基准
**Layer 3: 质量检查** — 详见下方清单
**Layer 4: 数据准确性** — "历史第X次""史上最大"等表述必须有精确来源

**终审检查清单：**
```
□ 文章 > 10KB
□ 6故事结构完整（或合理跳过）
□ 每条故事有数据+叙事+洞察
□ generate-charts.py 已运行且质量门禁 30/30 通过
□ 图表质量门禁 Layer 2.5 逐项通过（颜色/标签/数据/视觉/嵌入）
□ docs/charts/ 目录当天有更新
□ 嵌入的图表与对应板块叙事相关（非通用数据硬塞）
□ 所有漫画面板存在且 < 6KB
□ 漫画风格匹配基准（白底/黑线/300×220/无色块无人物）
□ 漫画无文字遮挡（文字在空白区域，不重叠）
□ 无"历史第X次"类可疑表述
□ 无"史上最大"类无来源表述
□ 无"值得关注""后市可期""需警惕"等套话（全文grep覆盖.html+.md）
□ script.txt 可用（无英文/标签问题）
□ script.txt 开口="朋友们早上好"（早报）/ "各位朋友晚上好"（晚报）
□ script.txt 结尾="我们晚上见"（早报）/ "明天早上见"（晚报）
□ script.txt 有晚报预告（早报必选）
□ audio.mp3 存在且>100KB
□ article.md 存在
□ 合规声明存在（"不构成投资建议"）
□ 无模板占位符残留（grep检查{{）
□ 数据索引已更新
□ 音频时长与页面标注差异≤20%
□ 脚本字数/预计时长/实际时长一致
```

**任何一条不通过 → 不发布，先修复。**

### Phase 7: 部署

```
1. python scripts/update-index.py（更新数据索引）
2. 验证 data/articles.json 包含最新文章
3. grep -n '{{' index.html article.html（检查无模板占位符）
4. git add → commit → push
5. 等待 2 分钟（GitHub Pages 构建部署）
6. 运行验证脚本（验证清单逐一检查）
7. 告知用户：已推送，约 1-2 分钟后生效
```

**部署验证清单（逐一检查）：**
```
□ curl index.html | grep data-date → data-date 为当日
□ curl articles.js | grep latest → latest_date 为当日
□ curl article.html → HTTP 200 且 > 10KB
□ curl vix-data.js | grep current → VIX 有数据
□ 无模板占位符残留（grep -n '{{'）
```

**关键禁令：**
- ❌ 禁止手动修改 index.html 的预渲染内容（JS 自动处理）
- ❌ 禁止运行 update-index.py 前不读代码（理解输入输出）
- ❌ 禁止移除任何函数/代码行除非100%确认其无用
- ❌ 禁止推送后不等待部署就断言"没更新"

## 晚报生产流程

### 三层固定结构（强制）

晚报文章必须严格遵循以下三层结构，不得省略任何一层：

```
┌─────────────────────────────────────┐
│ LAYER 1：今日热点速览（强制）        │
│ 标题: "📋 今日热点速览 — 60秒掌握   │
│       关键信息"                      │
│ 内容: panel-000.svg 热点速览漫画      │
│       （辐射状雷达图，6-7个核心事件）  │
│ 分隔: 虚线 + "▼ 往下翻看深度解读"     │
├─────────────────────────────────────┤
│ LAYER 2：今日深度专题（强制）        │
│ 副标题: "🔍 今日深度专题"            │
│ 引语: 金色左边框quote块，概括当日      │
│       核心矛盾/悬念                   │
│ 故事: 6个编号故事（01-06）            │
│       每故事含 head + content + img  │
│ 结尾: gold-box 金色签语句             │
│       "我是阿扬，我们明天早上见。"     │
├─────────────────────────────────────┤
│ LAYER 3：关键市场数据（强制）        │
│ 标题: "📊 关键市场数据一览"          │
│ 表格: 核心品种行情数据表              │
│ 列表: tag-b 标签化摘要（5-6条）       │
│ 来源: sources块 + 免责声明            │
│ 品牌: 金色品牌尾块 + warn风险提示     │
└─────────────────────────────────────┘
```

### 各层详细规范

**Layer 1 — 今日热点速览：**
- 样式参考 `2026-05-22/wechat-publish/evening/article.html`
- CSS类名：`.layer1` / `.l1-title` / `.l1-comic` / `.l1-divider`
- 漫画panel-000.svg必须是辐射状雷达图（中心hub + 放射线 + 6节点）
- 每个节点标注一个当日核心事件（标题8-10字+副标题）
- 不对每篇文章配单独漫画，这一张是"速览"而非"解读"

**Layer 2 — 今日深度专题：**
- 样式参考同上，使用 `.story` / `.story-head` / `.story-content` / `.story-img`
- 故事编号色标：
  ```
  01 A股 → #EF4444 (红)
  02 科技 → #7C3AED (紫)
  03 AI/产业 → #F59E0B (橙)
  04 国际 → #1A56DB (蓝)
  05 宏观 → #10B981 (绿)
  06 商品/黄金 → #D4A017 (金)
  ```
- 每个故事必须包含：story-num + story-tag + story-title + story-badge + story-content p
- 漫画按需嵌入 story-img，不固定数量
- 故事之间用 story-divider 分隔

**Layer 3 — 关键市场数据一览：**
- 样式参考同上，使用 `.layer3` / `.l3-title` / `table` / `.list`
- 数据表必须包含品种、最新价、涨跌幅三列
- list标签化摘要：每条以tag-b开头 + 简要描述
- 数据来源用 `.sources` 块
- 品牌尾块 + `warn` 风险提示

### 与早报的差异

| 维度 | 早报 | 晚报 |
|------|------|------|
| 核心 | 隔夜全球新鲜事件 | A股当日深度复盘 |
| 结构 | 自由段落 | 强制3层（速览+深度+数据） |
| 01 A股 | 简要数据回顾（1段） | 完整深度分析（独立故事） |
| 02-06 | 全球焦点展开 | 延续深度分析+明日展望 |
| 漫画 | panel-000起速览 | panel-000必为雷达速览图 |
| 结尾 | 端块结束 | gold-box签语句 + Layer3数据表 |
| 用户期待 | 晨间快速了解发生了什么 | 晚间理解为什么发生 |

## 10条铁律（摘自 TEAM.md v5.1）

1. **数据不准，整篇重做** — 每个数据点必须追溯来源
2. **标多少就做多少** — 音频时长/字数/阅读时间标多少就是多少
3. **风格不可随意变更** — 每次生产前先读风格样本，变更必须先展示预览
4. **模板占位符不能残留** — HTML中不能出现`{{`（CSS content除外）
5. **用户反馈不能忽视** — 连续2次同一方向错误 → 停下来追问
6. **未确认不新增元素** — 任何新增内容/视觉元素/功能模块，先征求意见
7. **使用已有工具链** — generate-charts.py等脚本不能跳过
8. **用户同问题≥2次 → 冻结专攻** — 暂停其他所有工作，专攻此问题
9. **JS变更必须浏览器验证** — 不得"先推了再说"
10. **推送前必须执行部署验证清单** — curl逐一检查后通知用户
