# 扬说财经 · 生产工作流程 v4.0

> 版本: v4.0 | 更新: 2026-05-22
> 核心变更: 6故事分类法 + 用户确认门禁 + Codex协作升级 + 质量检查升级

---

## 核心原则

- **6故事固定分类** — 每天内容按01-06编号展开，顺序固定
- **数据+叙事+洞察** — 每条故事必须覆盖三维度，缺一不可
- **专职核查** — 事实核查员独立运作，可阻塞流水线
- **独立质检** — 质检审查官不参与生产，专门找问题
- **先假设后验证** — 每个Agent启动输出假设，完成输出证据
- **早晚分离** — 早报和晚报是两次独立的生产调用
- **日报读晚报** — 生产早报前必须检查前日晚报内容，确保Layer 2不重复
- **用户确认门禁** — 重大变更必须先展示预览，用户确认后执行
- **Codex协作** — SVG生成委托Codex，Claude Code负责审查
- **每日复盘** — 问题不过夜，每条问题落实到下轮生产

---

## 五层质量控制体系（v4.0升级）

```
Layer 1: Agent 1（新闻采集员）→ 数据源头质量
Layer 2: Agent 2（事实核查员）→ 数据来源追溯 ← 独立门禁，可阻塞
Layer 3: Agent 3/4/5（生产者）→ 内容 + SVG + 音频自检
Layer 4: Agent 7（质检审查官）→ 客观证据收集 ← 独立门禁，可驳回
Layer 5: Claude Code（主编统筹）+ 用户确认 → 终审门禁
```

**v4.0新增质检维度：**
- 6故事结构完整性检查
- 每故事三维度覆盖检查（数据+叙事+洞察）
- 空洞套话扫描
- 素描风格合规检查
- 口播稿与文章一致性检查

## 早报生产流程（11阶段，v4.0升级版）

### Phase 0: 环境验证（主编）
```
Think 假设:
1. [工具链可用性假设]
2. [数据源可用性假设]
3. [前日晚报已检查] — 确保Layer 2不重复

验证:
→ python --version
→ git status (仓库干净)
→ 生产目录就绪
→ edge-tts 可用
```

### Phase 1: 新闻采集（Agent 1 / Claude Code）
按6故事分类搜索隔夜全球财经新闻：
- 01 A股 → 昨日行情数据
- 02 美股 → 隔夜收盘数据
- 03 科技 → 产业动态
- 04 国内政策 → 政策突发
- 05 宏观 → 央行/债市/美联储
- 06 商品 → 黄金/能源

**输出:** `docs/review/news-morning-{YYYY-MM-DD}.md`
**完成后输出验证证据块。**

### Phase 2: 事实核查（Agent 2）
逐条验证新闻的数据点来源，重点核查：
- 所有百分比数字有来源
- 所有价格数字标注市场和时间
- 无"历史第X次"表述
- 关键数据至少有2个独立来源

**发现编造数据 → 阻塞流水线。**

### Phase 3: 结构设计（Claude Code + 用户确认 ← 新增）
```
【新增步骤】设计文章结构草稿，展示给用户确认:
1. 6故事标题+各故事的数据点
2. 6个SVG的视觉隐喻设计
3. 金句收尾

等待用户反馈 → 确认后进入Phase 4
```

**用户连续2次指出同一方向错误 → 停下来追问，不再猜测。**

### Phase 4: 文章撰写（Agent 3 / Claude Code）
写入 article.html，遵循6故事结构。
- Layer 1: 热点速览（panel-000）
- Layer 2: 6个编号故事（panel-001~006，每故事配SVG）
- Layer 3: 数据表 + 图表

**强制要求：** 每条故事必须包含数据+叙事+洞察三维度。

### Phase 5: SVG设计（Claude Code 设计 → Codex CLI 生成 ← 新增协作）
1. **Claude Code 设计**：输出每个SVG的视觉隐喻描述、布局草图、色彩方案
2. **Codex CLI 生成**：批量生成 panel-000 ~ panel-006 的SVG代码
3. **Claude Code 审查**：逐条检查素描风格合规性，不符合的标记退回

**审查清单：**
- [ ] 线稿为主（非信息图）
- [ ] 色彩不超过3种
- [ ] 视觉隐喻清晰
- [ ] 文件 < 30KB
- [ ] 无渐变/阴影

### Phase 6: 音频制作（Agent 5）
1. 写口播稿 script.txt：遵循6故事结构，口语化表达
2. 主编审核
3. 合成 audio.mp3

**v4.0新要求：** 口播稿必须与文章内容对齐（故事顺序、关键数据一致）

### Phase 7: 质量审查（Agent 7 / Claude Code）
独立执行质检，收集客观证据。

**检查清单（v4.0升级版）：**
- [ ] 6故事结构完整（01-06）
- [ ] 每故事含数据+叙事+洞察
- [ ] 无空洞套话（"值得关注""后市可期"等）
- [ ] 无"历史第X次"类表述
- [ ] SVG为素描风格
- [ ] 文件大小合规
- [ ] 合规声明存在
- [ ] 口播稿与文章一致

**全部通过才放行，有问题直接驳回。**

### Phase 8: 用户预览（新增门禁）
```
展示预览内容给用户:
1. 最终文章HTML预览
2. 新版SVG各面板效果
3. 口播稿

等待用户确认 → 确认后进入Phase 9
用户不满意 → 回到对应Phase修改 → 重新审查 → 重新预览
```

### Phase 9: 部署（Agent 8）
```
git add <修改文件>
git commit -m "feat: 扬说财经早报 {日期} — {改动要点}"
git push origin main
验证URL可访问
```

### Phase 10: 复盘（Agent 9）
当日总结 → 改进项记录。
输出: `docs/review/daily-{YYYY-MM-DD}.md`

## 专题生产流程（6天周期）

### 核心原则

- **日更优先级高于专题** — 专题不得影响日更早报/晚报的正常发布
- **用户确认门禁** — 结构设计阶段必须展示预览，用户确认后执行
- **三种内容方向轮换** — A热点深度/B知识科普/C独到观点 = 3:3:2
- **6工作日生产周期** — 周一选题决策 → 周六发布

### 生产周期流水线

```
周一（Day 1）— 选题决策
├── S-01 从选题池筛选3-5个候选选题
├── S-01 输出选题评估卡（难度/素材/风险/角度）
├── 主编确定选题
├── 用户确认选题（第1道门禁）
└── 输出: docs/review/topic-preview-{topic}.md

周二~周三（Day 2-3）— 深度研究
├── S-02 执行深度采集
├── 覆盖范围：官方数据/权威媒体/行业报告/学术来源
├── 关键数据双源交叉验证
├── 输出SVG视觉隐喻建议
└── 输出: docs/review/research-{topic}.md

周四（Day 4）— 结构设计 + 用户确认
├── S-03 基于研究简报输出文章大纲
│   ├── 3-4节结构草案
│   ├── 各节核心数据和观点
│   └── 每节对应的SVG视觉隐喻设计
├── S-04 输出SVG设计稿（视觉描述）
├── 展示给用户确认（第2道门禁）
└── 用户确认 → 进入Day 5；不满意 → 修改

周五（Day 5）— 集中生产
├── 上午: S-03 撰写文章正文
│   └── 使用 special/.templates/article-template.html
├── 下午: 
│   ├── S-04 / Codex CLI 批量生成SVG
│   ├── S-05 编写口播稿 → script.txt
│   ├── S-05 合成音频 → audio.mp3 (edge-tts)
│   └── C-07 执行质量审查（专题专用检查项）
├── 质检不通过 → 退回对应Agent修改
└── 质检通过 → 进入Day 6

周六（Day 6 8:00 AM）— 发布
├── C-08 部署
│   ├── python scripts/update-index.py
│   ├── git add → commit → push
│   └── 验证URL可访问
├── S-01 更新选题池状态（⚪→🟢）
└── S-01 输出复盘（阅读量/收藏率/改进项）

周六~周日 — 数据沉淀
├── 收集阅读量/收藏量/分享率
├── 记录用户评论和反馈
└── 更新 topic-pool.md 数据沉淀表
```

### 质量检查清单（C-07 专题专用）

在日更检查项之外，新增以下检查：

```
专题文章专用检查项：
- [ ] 文章2000-4000字
- [ ] 3-4节结构完整
- [ ] 每节配1张SVG漫画
- [ ] 每500字有停顿节点
- [ ] 术语有解释（附术语表）
- [ ] 有"反方观点"或"不确定性"段落
- [ ] 数据来源标注格式统一
- [ ] 引用报告有名称+日期+发布机构
- [ ] SVG是机制/流程可视化（非仅数据展示）
- [ ] 口播稿8-15分钟长度
- [ ] 无"我预测""我肯定"等武断表述
- [ ] 无投资建议（不推荐个股）
```
