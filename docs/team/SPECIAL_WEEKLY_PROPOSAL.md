## 一、为什么需要周更专题？

### 当前短板

| 维度 | 日更（早报/晚报） | 周更专题 |
|------|------------------|---------|
| 时间纵深 | 当天发生什么 | 一个主题的系统拆解 |
| 知识沉淀 | 看完即弃 | 可收藏、可搜索、可系列化 |
| 用户粘性 | 信息消费 | 认知提升 → 品牌认同 |
| 差异化 | 各家都有 | "只有扬说才能讲清楚" |
| SEO/搜索流量 | 几乎无 | 长尾流量来源 |

**一句话定位：** 日更新闻帮你"知道发生了什么"，周更专题帮你"理解为什么会这样、接下来可能怎样"。

## 二、栏目设计

### 2.1 命名与品牌

**栏目名：扬说·深度**（英文：Yangshen Insight）

子品牌视觉规范：
- Logo: 现有"扬说财经"品牌下，加 **"深度"** 二字（金色 #D4A017）
- 固定栏目头：每篇文章顶部插入统一SVG横幅标识
- 标签体系：#扬说深度 + 主题标签
- Slogan: "用逻辑穿透表象"

### 2.2 更新节奏

| 时间 | 内容 |
|------|------|
| 周一~周五 | 早报 + 晚报（现有不变） |
| **周六** | **专题发布（8:00 AM）** |
| 周日 | 停更（或应急补发） |

> **为什么选周六？** 周五全球市场收盘后，有周末时间消化；用户周末有阅读时间；避开幕前焦虑。

### 2.3 内容框架（黄金模板）

每篇深度专题遵循以下统一结构：

```
┌─────────────────────────────────────┐
│ 1. 封面标题 + 引语（100字）           │
│    用一句核心判断/悬念抓住读者          │
├─────────────────────────────────────┤
│ 2. 开头：抛出一个问题（200字）          │
│    "你有没有想过，为什么……"           │
│    用反常识/被忽视的现象切入            │
├─────────────────────────────────────┤
│ 3. 正文：分层拆解（3~4节，每节400-600字）│
│                                   │
│   §1 发生了什么 — 数据/事实/案例     │
│   §2 为什么会这样 — 逻辑/机制/推演    │
│   §3 意味着什么 — 影响/趋势/判断      │
│   §4 接下来可能怎样 — 展望/风险/建议   │
│                                   │
│   每节配1张SVG漫画（辅助理解）         │
├─────────────────────────────────────┤
│ 4. 结尾：一句话总结 + 行动建议（150字）│
│    有力收尾，留下思考空间               │
├─────────────────────────────────────┤
│ 5. 附录                          │
│    - 关键数据表                     │
│    - 术语解释（glossary）           │
│    - 参考来源清单                    │
│    - "如果你还想了解……"推荐阅读       │
├─────────────────────────────────────┤
│ 6. 音频版（8-15分钟）                │
│    - 完整朗读 + 口语化改编            │
│    - 含"阿扬说"个人观点段落           │
└─────────────────────────────────────┘
```

### 2.4 三种内容方向（轮换制）

按 **3:3:2** 比例分配（每季度约12期）：

| 方向 | 季度占比 | 目标 | 难度 | 示例选题 |
|------|---------|------|------|---------|
| A. 热点深度解读 | ~5期 | 传播力 + 搜索流量 | ⭐⭐ | 《英伟达财报为什么能影响全球科技股？》 |
| B. 知识科普系列 | ~5期 | 用户收藏 + 长期沉淀 | ⭐⭐⭐ | 《普通人需要知道的10个经济指标》系列 |
| C. 独到观点洞察 | ~2期 | 品牌调性 + 铁杆粉丝 | ⭐⭐⭐⭐ | 《为什么说AI硬件的真正机会不在芯片》 |

> **选题优先级原则：** 热点 > 系列 > 观点。新人先做A，成熟后做B，有积累后做C。

### 4.4 Agent S-03: 专题主笔

**写作规范（日更基础上的升级）：**

| 维度 | 日更标准 | 专题升级 |
|------|---------|---------|
| 篇幅 | 800-1200字 | **2000-4000字** |
| 叙事层数 | 数据+叙事+洞察 | 数据+叙事+洞察+展望+反方 |
| 数据精度 | 小数点2位 | + 时间维度对比（同比/环比/历史分位） |
| 来源引用 | URL | + 报告中引用（格式统一） |
| 术语处理 | 避免 | 使用+解释（附术语表） |
| 观点态度 | 有态度但不武断 | + 承认反对意见+说清为什么不同意 |
| 可读性 | 分段+小标题 | + 加粗关键句 + 每500字一个"停顿节点" |

**强制写作原则：**
1. **不堆砌术语** — 每个术语必须解释，假设读者是"聪明的高中生"
2. **先说人话再说专业话** — 先用比喻让读者理解，再用术语精确表达
3. **每个观点必须有依据** — 不发表没有来源支撑的判断
4. **承认不确定性** — "从目前情况看，更可能……而不是……"
5. **用"我们"不用"你们"** — "我们普通人需要关注的是……"
6. **每500字一个停顿节点** — 一个提问/一个比喻/一个"有意思的是"打断平铺直叙

**完成输出：** `special/{topic-slug}/article.html`

### 4.5 Agent S-04: 可视化设计师

**与日更SVG的根本区别：**

| 维度 | 日更SVG | 专题SVG |
|------|---------|---------|
| 数量 | 6-7个 | **3-5个** |
| 内容 | 数据展示 | **流程/机制/链路的可视化** |
| 复杂度 | 单层信息 | **多层嵌套（因果链/时间线/对比）** |
| 风格 | 素描风一致 | 素描风一致，但可加入**流程箭头/表格/示意** |
| 角色 | 无角色 | **可加入阿扬角色/对话气泡** |

**专题SVG类型：**
1. **流程/机制图** — "美联储加息如何传导到你的房贷"（因果链可视化）
2. **对比/决策树** — "不同经济环境下的资产配置"  
3. **时间线/演变** — "芯片从设计到量产的全流程"
4. **数据可视化** — "过去10年金价与美元指数的关系"（简化的散点/折线）

**审查清单（日更基础上新增）：**
- [ ] 流程正确（无逻辑错误）
- [ ] 箭头方向正确
- [ ] 术语标注准确
- [ ] 无信息过载（一张SVG只讲一个逻辑）

| **专题流水线** | 策划研究期 | **发布日** | 复盘 |

> **关键约束：** 周更专题不得影响日更早报/晚报的正常发布。日更优先级高于专题。

## 九、风险控制

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| 选题重复/与其他媒体撞车 | 中 | 中 | 发前搜索同类标题，确保角度不同 |
| 数据过时（周末市场变化） | 中 | 高 | 周六发布前重新验证关键数据 |
| 观点偏差引发争议 | 低 | 高 | C类选题提前评审，标注"个人观点" |
| 用户不感兴趣 | 中 | 中 | 前3期免费试水，收集反馈 |
| 影响日更质量 | 低 | 高 | 严格遵守"日更优先级高于专题"原则 |
| 涉及敏感话题 | 低 | 高 | 选题评估卡增加风险等级字段 |

## 十、第一期建议

作为栏目的"开场秀"，建议首期选择 **B类（知识科普）** 方向，原因是：
1. 风险最低 — 不涉及市场判断
2. 门槛最低 — 不需要高度时效性
3. 最能体现"扬说"风格 — 用漫画讲清楚一个概念
4. 最适合测试流程 — 可以从容完成全流程测试

**推荐的第一个选题：**
> **《什么是美债收益率？为什么它"倒挂"全世界都紧张？》**
>
> 理由：近期美债收益率反复波动，关注度高；适合用流程图+漫画可视化；可以标注"读懂宏观系列·第1期"，为后续做系列打基础。
