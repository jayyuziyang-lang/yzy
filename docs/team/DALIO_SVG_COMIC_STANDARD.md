## 一、设计哲学（为什么这样做）

Ray Dalio 的动画用30分钟讲清楚了宏观经济如何运行。它不是靠画得好看，而是靠三个设计原则：

| 原则 | Dalio 的做法 | 我们的做法 |
|------|-------------|-----------|
| **极简** | 白纸背景 + 3-4个元素 | SVG 白色/浅灰底 `#FAFBFC`，每图 ≤ 5个视觉元素 |
| **符号化** | 买方/卖方/银行 = 简笔画小人 + 标签 | stick figure 小人 + 物件（工厂/火箭/天平/温度计） |
| **标签化** | 每个物件标注经济含义 | 中文标签直接写在经济物件上或旁边 |

**关键判断：复杂概念 → 简单视觉 → 读者秒懂。** 如果一张漫画需要5秒以上才能理解，它失败了。

## 二、标准视觉语言

### 2.1 画布规格

| 参数 | 值 |
|------|-----|
| 尺寸 | 768 × 512 (px) |
| 背景色 | `#FAFBFC`（暖白） |
| 字体 | `Microsoft YaHei, SimHei, sans-serif` |
| 输出格式 | SVG（矢量，2-6KB） |

### 2.2 配色方案

**配色比例系统（强制）：**
- 60% 白色/浅灰背景
- 20% 蓝色主色（金融系统/稳定性/信用）
- 10% 橙色/金色强调色（交易/现金流/经济活动）
- 5-10% 红色/绿色功能色（风险/增长）

| 用途 | 色值 | 说明 |
|------|------|------|
| 暖白背景 | `#FAFBFC` | 清晰、教育感 |
| 品牌蓝（银行/稳定/信用） | `#1A56DB` | 早报统一色 |
| 深蓝（文字/标题） | `#1E40AF` | 蓝底白字标题 |
| 正红（上涨/风险/热度） | `#DC2626` | 涨=红，全球市场统一 |
| 正绿（下跌/利好） | `#16A34A` | 跌=绿 |
| 警示红（风险/关门/危机） | `#EF4444` | 柔和红 |
| 暖金（黄金/强调/商品） | `#D4A017` / `#F59E0B` | 商品板块专用 |
| 橙色（石油/能源/消费） | `#EA580C` | 商品能源 |
| 紫色（科技/AI） | `#7C3AED` | 科技板块专用 |
| 深灰（线条/正文） | `#475569` | 主线条 |
| 浅灰（辅助线/虚线） | `#94A3B8` / `#CBD5E1` | 辅助元素 |
| 浅蓝（背景区） | `#EFF6FF` | 卡片背景 |

**禁止使用：** 大面积霓虹色、过度渐变、复杂纹理、暗黑科技风、高饱和杂乱配色

### 2.3 核心视觉组件库 v2.0

每个组件都是可复用的 Python 函数：

```python
# 人物（几何形状——圆头+梯形身体）
geo_person(x, y, h=44, color="#1A56DB", role="stand")
# role: stand / arms_up / look_up / point / carry

# 文字标注
label(x, y, text, size=13, color="#334155", anchor="middle", bold=False, bg=None)

# 箭头
arrow(x1, y1, x2, y2, color, marker, sw=2.5)

# 虚辅助线
dashed_line(x1, y1, x2, y2, color="#CBD5E1")

# 齿轮（经济机器/传动）
gear(cx, cy, r, color="#1A56DB", teeth=8)

# 旋钮/调节器（利率/政策）
knob(cx, cy, r, angle=0, color="#1A56DB")

# 管道+阀门（信贷阀门/资金流动）
valve_pipe(cx, cy, w, h, color, open_valve=True)

# 温度计（经济热度/通胀）
thermometer(cx, cy, h, temp_pct, color="#DC2626")

# 水池（资金池/流动性——fill_level 0-100）
pool(x, y, w, h, fill_level, color, label_text, sub_text)

# 时钟（等待/倒计时）
clock_face(cx, cy, r, hour_angle=0)

# 天平（多空/利弊平衡）
scale(x, y, tilt_deg=0, l_weight=False, r_weight=False)

# 气球（泡沫）
balloon(x, y, r, expanding=False, color="#DC2626")

# 工厂（制造业/生产）
factory(x, y, w, h, color, smoking=True)

# 关门店（消费低迷）
store_closed(x, y, w, h)

# 标题栏
title_bar(text, color="#1A56DB")

# 底部注释
footnote(text, y=500)
```

**geo_person 六种姿态：**
- `"stand"` — 正常站立（观望/等待）
- `"arms_up"` — 举手欢呼（牛市/赚钱）
- `"look_up"` — 仰望（看股价/看骰子/期待）
- `"point"` — 指向（引导视线/强调方向）
- `"carry"` — 搬运（劳动/生产/负重）

**人物类型：** 消费者、工人、企业主、银行职员、央行代表、投资者、房主、借款人、储户、商家

### 2.4 常用物件模板

| 物件 | 用途 | 构建方式 |
|------|------|---------|
| 工厂 | 制造业/生产 | 矩形+烟囱矩形+圆烟圈 |
| 商场 | 消费/零售 | 矩形+卷帘门横线+✕标志 |
| 铜牛 | 美股牛市 | 椭圆身体+圆头+角+腿 |
| 火箭 | 股价/市值飙升 | 椭圆箭体+三角头+火焰三角 |
| 天平 | 多空力量对比 | 底座矩形+竖线+横梁+圆弧托盘 |
| 温度计 | 经济热度对比 | 圆角矩形+球+液柱 |
| 骰子 | 不确定性/概率 | 三面等距多边形+?号 |
| 悬崖 | 风险/下跌 | 地面线突然截断+裂缝 |
| 皮筋 | 估值-基本面张力 | 贝塞尔曲线 `stroke-dasharray` |

## 三、隐喻设计框架（每次生产前必过）

### 3.1 三步设计法

```
第1步：问"这个经济故事的核心矛盾是什么？"
       → 不能用抽象词（"结构性矛盾"），要用具体词（"工厂冒烟但商场关门"）

第2步：找"什么日常物件能表达这个矛盾？"
       → 温差 → 温度计
       → 失衡 → 天平
       → 脱节 → 皮筋/锚链
       → 风险 → 悬崖
       → 不确定性 → 骰子

第3步：画草图（在心中或纸上）
       → 左边是什么？右边是什么？中间是什么？
       → 小人在哪？他们在做什么动作？
       → 哪些位置需要文字标注？
```

### 3.2 隐喻质量自检

```
□ 这个隐喻能用一句话讲清楚吗？（10岁小孩能听懂吗？）
□ 画面里有具体物件吗？（不是抽象图形）
□ 颜色有对照含义吗？（红=热/涨/风险，蓝=冷/跌/安全）
□ 读者能在3秒内理解吗？
```

## 四、生产流程（每张漫画6步）

### Step 1: 读文章找画点

从当天 article.html 中提取每个故事的核心矛盾。不是每个故事都需要配图——只画**有核心矛盾张力**的故事。

```
01 A股 → 什么和什么在对抗？
02 美股 → 什么风险被忽视了？
03 科技 → 什么数字脱离了什么现实？
...
```

### Step 2: 设计视觉隐喻

用三、3.1的三步法，为每个需要配图的故事设计一个具体隐喻。

### Step 3: 编写 Python SVG 生成函数

写一个 `comic_0X()` 函数，使用标准组件库（小人、物件、标签、箭头），构建场景。

### Step 4: 嵌套到生成脚本

将各 `comic_0X()` 函数嵌入 `gen-dalio-comics.py`（或当日对应的生成脚本），统一运行。

### Step 5: 质量自检（6项强制）

```
□ [XML语法]    Python xml.etree.ElementTree.parse() 通过
□ [font-family] <svg> 根标签有 font-family="Microsoft YaHei, SimHei, PingFang SC, sans-serif"
□ [&转义]     所有&必须是&amp;（无裸露的&）
□ [尺寸]       viewBox="0 0 768 512"，width >= 700
□ [颜色]       红涨绿跌：涨→#DC2626，跌→#16A34A（所有市场统一）
□ [文件大小]   尽量 < 12KB（简化SVG通常在4-12KB）
```

### Step 6: 嵌入 article.html

用 `<img src="comic/panel-0X.svg">` 嵌入对应 story 区块。

## 五、日报 vs 专题差异

| 维度 | 日报（早报/晚报） | 专题（扬说深度） |
|------|-------------------|-----------------|
| 配图数量 | 3-6张（按需） | 4-6张（每节1张） |
| 单图复杂度 | 2-3个视觉元素 | 3-5个视觉元素 |
| 配色 | 早报蓝 `#1A56DB` + 通用色 | 可配合专题主题色 |
| 文字标注 | 标题 + 2-3行副标题 | 标题 + 多行分层标注 |
| 设计重点 | 3秒秒懂 | 配合深度叙事，可多停留 |
| 角色 | 简笔画小人（符号化） | 同，可加入更多姿态变化 |

## 六、禁止事项

- ❌ 禁止用 Stable Diffusion 生成漫画（风格不可控、概念不精确）
- ❌ 禁止画抽象概念图（没有具体物件的"意境图"）
- ❌ 禁止使用深色背景
- ❌ 禁止一张图超过 8 个视觉元素
- ❌ 禁止文字和图形重叠
- ❌ 禁止凭记忆设计隐喻——必须先读完当天文章
