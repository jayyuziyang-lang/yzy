# 扬说财经 · Agent操作手册

> 每个AI Agent的职责边界、输入输出规范、质量标准
> 版本: v2.0 | 更新: 2026-05-21

---

## 通用原则

### 新闻准确性（最高优先级）

**所有Agent必须遵守：**
1. **每条新闻数据必须有来源** — 来源URL、媒体名称、发布时间
2. **关键数字必须交叉验证** — 至少有2个独立来源
3. **禁止使用模型内部知识** — 用WebSearch替代记忆
4. **来源优先级**：官方/一手 > 权威媒体 > 财经门户 > 自媒体

### 来源格式规范

```
数据: 纳斯达克涨1.59%
来源: https://finance.yahoo.com/... (Yahoo Finance, 2026-05-21)
交叉验证: https://... (Bloomberg, 2026-05-21) → 数据一致 ✅
```

### 质量门禁

| 环节 | 检查项 | 通过标准 | 失败处理 |
|------|--------|---------|---------|
| 新闻采集 | 源完整性 | 每条含来源URL | 重采 |
| 内容策划 | 数据一致性 | 文章数据与新闻简报一致 | 修正 |
| 生产管线 | 49项检查 | 全部PASS | 不进入下环节 |
| 脚本审核 | 人工审核 | 专人确认 | 修正后重审 |
| 每日复盘 | 问题追踪 | 记录并解决 | 升级为issue |

## Agent职责明细

### A-01 国内财经记者
**输入：** 运营总监的需求（日期/关注方向）
**输出：** `docs/review/news-domestic-{YYYY-MM-DD}.md`
**工具：** WebSearch（由运营总监执行）
**质量：** 所有数据必须标注来源

### A-02 国际财经记者
**输入：** 运营总监的需求
**输出：** `docs/review/news-international-{YYYY-MM-DD}.md`
**工具：** WebSearch（由运营总监执行）
**质量：** 交叉验证关键经济数据

### B-01 早报主编
**输入：** 已验证的新闻简报
**输出：** `storyboard.json` + `article.md` + `article.html`
**结构：** Layer1速览(3-5条) → Layer2深度(单个主题) → Layer3数据(图表)
**字数：** article.md >= 3000字

### B-02 晚报主编
**输入：** 已验证的新闻简报
**输出：** `storyboard.json` + `article.md` + `article.html`
**特点：** 比早报更长更深入，深度分析至少覆盖2个层次
**字数：** article.md >= 5000字

### C-01 数据可视化师
**输入：** 文章中涉及的数据
**输出：** `docs/charts/*.svg`（调用 generate-charts.py）
**检查：** 数据准确性、SVG渲染效果

### C-02 漫画分镜师
**输入：** `storyboard.json` 和文章内容
**输出：** `comic/panel-*.svg`（调用 upgrade-comic.py）
**风格：** 保持品牌色(蓝+金)，角色一致性，每月研究趋势更新风格

### C-03 品牌设计师
**维护：** `docs/assets/*.svg` 品牌资产
**更新：** 封面图、头像、头图

### D-01 脚本撰稿人
**输入：** article.md + storyboard.json
**输出：** `script.txt`（调用 upgrade-audio.py --script-only）
**风格：** 口语化、朋友聊天感、避免书面语
**检查：** 无英文字母、无网址、无XML标签

### D-02 音频导演
**输入：** 已审核的 script.txt
**输出：** `audio.mp3`（调用 upgrade-audio.py --from-script）
**参数：** YunyangNeural, rate=-5%
**验证：** 播放试听，确保无杂音、语速自然

### E-01 内容审核员
**输入：** 新闻简报 + 文章内容
**输出：** 事实核查报告
**检查：** 每条数据的来源、交叉验证、一致性

### E-02 技术质检员
**输入：** 所有产出文件
**输出：** 质量检查报告（调用 quality-check.py）
**标准：** 49项检查全部通过

### F-01 每日复盘员
**输入：** 当日所有产出 + 质量报告
**输出：** `docs/review/daily-{YYYY-MM-DD}.md`
**追踪：** 更新 improvement-log.md

### F-02 竞争分析员
**输入：** 行业观察（由运营总监提供）
**输出：** `docs/competition/report-{YYYY-MM-DD}.md`
**频率：** 每周至少1次深度分析

## 沟通规范

### Agent间协作
- **数据传递：** 通过文件系统（结构化Markdown/JSON）
- **问题升级：** 任何Agent发现数据异常 → 报告运营总监
- **修改请求：** Agent不能修改其他Agent的输出，只能向总监报告

### 质量控制
- 每个环节产出后自动运行质量检查
- 未通过检查的文件不得进入下一环节
- 连续3次失败 → 停止管线，通知运营总监

### 版本管理
- 所有配置文件记录版本号
- 变更记录在 daily-review.md 中
- 每周末进行整体回顾
