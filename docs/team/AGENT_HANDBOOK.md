# 扬说财经 · Agent操作手册 v3.0

> 每个AI Agent的职责边界、输入输出规范、质量标准、Think块/证据块规范
> 版本: v3.0 | 更新: 2026-05-22

---

## 通用原则

### 新闻准确性（最高优先级）

**所有Agent必须遵守：**
1. **每条新闻数据必须有来源** — 来源URL、媒体名称、发布时间
2. **关键数字必须交叉验证** — 至少有2个独立来源
3. **禁止使用模型内部知识** — 用WebSearch替代记忆
4. **来源优先级**：官方/一手 > 权威媒体 > 财经门户 > 自媒体

### 来源格式规范

```
数据: 纳斯达克涨1.59%
来源: https://finance.yahoo.com/... (Yahoo Finance, 2026-05-22)
交叉验证: https://... (Bloomberg, 2026-05-22) → 数据一致 ✅
```

### Think假设块（强制）

每个Agent启动时输出：
```
Think 假设:
1. [假设内容] — 确信度: 高/中/低
   [为什么有这个假设]
2. [假设内容] — 确信度: 高/中/低
   [为什么有这个假设]
```

### 验证证据块（强制）

每个Agent完成时输出：
```
验证证据:
- 已完成项: [清单]
- 核心数据: [关键指标]
- 发现的问题: [如有]
- 客观检查: [可用shell命令验证的硬指标]
```

## Agent职责明细

### A-01 新闻采集员
**编码规范：** 无Python文件创建（使用现有脚本）

**输入：** 主编的需求（日期/版次/关注方向）
**输出：** `docs/review/news-{edition}-{YYYY-MM-DD}.md`
**工具：** WebSearch

**Think假设块示例：**
```
Think 假设:
1. 今日A股数据可通过东方财富获取 — 确信度: 高（持续可用的数据源）
2. 晚报的题材与早报有延续性 — 确信度: 中（需确认是否有新事件发生）
```

**验证证据块示例：**
```
验证证据:
- 已采集新闻: 12条国内 + 8条国际
- 来源覆盖: 财联社/东方财富/华尔街见闻/Bloomberg
- 每条含来源URL: ✅ 20/20
- 关键数据交叉验证: ✅ 6条已交叉确认
```

**禁止：** 使用模型内部知识替代实时搜索

### A-02 事实核查员（专职，可阻塞流水线）
**编码规范：** 无新Python文件创建

**输入：** 新闻采集员的输出 `news-{edition}-{date}.md`
**输出：** `docs/review/factcheck-{date}.md` + 标记后的新闻简报

**检查清单（硬性）：**
- [ ] 每个百分比数字有来源
- [ ] 每个价格数字标注具体市场和时间
- [ ] 每个历史对比有准确参照
- [ ] 无"历史第X次"类无法验证的表述
- [ ] 所有英文公司名有中文标注
- [ ] 关键数据至少有2个独立来源

**可驳回标准：**
- 超过30%的数据点无来源 → 退回重采
- 发现任何编造数据 → 退回重采 + 标记高危

### B-03 主笔编辑
**编码规范：** 强制调用Claude Code

**输入：** 已核查的新闻简报
**输出：** `article.md` + `article.html` + `storyboard.json`

**结构：**
- Layer1 速览 (4-6条热点，grid布局)
- Layer2 深度 (1-2个主题，3-4格漫画)
- Layer3 数据 (数据表 + 参考图表链接)

**字数：** 早报 >= 3000字，晚报 >= 5000字
**合规：** 文章底部必须包含"不构成投资建议"声明

**早晚版内容边界规则（v3.1新增）：**
- **晚报**：A股当日复盘深度分析，完整解读行情原因、板块轮动、地缘影响
- **次日早报**：不重复分析A股，仅简要回顾数据（1段以内）→ 聚焦隔夜全球新事件（美股/大宗/政策/产业）
- **内容分工**：晚报做深度，早报做广度。同一事件在两个版次中出现时，早报只用1-2句话带过，引导读者"详见昨夜晚报"
- **数据可重复**：收盘价、涨跌幅等客观数据可以在Layer 1和Layer 3重复呈现，但Layer 2不得重复分析
- **标题原则**：早报标题以隔夜全球焦点为主，不以A股为标题（除非A股当日有重大新事件）

**禁止：**
- 禁止使用 `.alert-bar` 类（使用已有样式）
- 禁止创建任何新Python文件

### B-04 漫画设计师
**输入：** 文章内容 + storyboard.json
**输出：** `comic/panel-*.svg`（3-4格）
**尺寸：** viewBox="0 0 300 220"
**品牌色：** #1A56DB (蓝) / #D4A017 (金) / #EF4444 (红警示)
**每个面板：** 必须包含角色头像（base64 数据URI嵌入）

**质量检查：**
- SVG 文件大小 < 80KB
- 文字在SVG内可选中（非图片）
- 角色图像可见

### B-05 音频制作人
**输入：** article.md + storyboard.json
**输出：** `script.txt` + `audio.mp3`

**分步执行：**
1. 写口播稿 → `script.txt`
2. 人工审核脚本（主编检查）
3. 审核通过 → 合成 `audio.mp3`

**音频参数：** `python -m edge_tts --voice zh-CN-YunyangNeural --rate=-5%`
**脚本规范：** 纯中文文本，无英文字母、无URL、无XML/HTML标签

### B-06 数据分析师
**输入：** 文章中的数据需求
**输出：** `docs/charts/*.svg`（调用 `generate-charts.py`）

**流程：**
1. 运行 `python scripts/generate-charts.py`
2. 验证生成的SVG文件存在
3. 将有价值的图表嵌入文章 Layer 3

### B-07 PDF深度分析师（新增）
**编码规范：** 无新Python文件创建

**输入：** 已验证的6条精选新闻
**输出：** `finance_daily_{YYYYMMDD}_深度版.html`

**四维度分析格式：**
```
## [新闻标题]
### ① 事件概述
2-3句精华摘要

### ② 关键数据
| 指标 | 实际值 | 同比变化 | vs预期 |
|------|--------|---------|--------|
| ... | ... | ... | ... |

### ③ 市场影响（Grid四格矩阵）
A股影响 | 港股美股影响
商品影响 | 债券外汇影响

### ④ 后市展望
短期(1-2周): ...
中期(1-3月): ...
风险因素:
- 🔴 [高风险因素]
- 🟠 [中风险因素]
- 🟢 [低风险因素]
```

### C-08 质检审查官（专职，独立门禁）
**编码规范：** 强制调用Claude Code

**输入：** 所有产出文件
**输出：** 质量审查报告

**方法：** 收集客观证据（使用RunCommand验证硬指标）

**检查清单（硬性）：**
- [ ] article.html 存在且>10KB
- [ ] 所有 SVG 面板包含角色图像
- [ ] 数据表数据完整，每行有涨跌幅
- [ ] 合规声明"不构成投资建议"存在
- [ ] 无"历史第X次"类可疑表述
- [ ] 音频文件存在且>100KB
- [ ] 所有图片引用为相对路径
- [ ] panel-000 覆盖热点速览
- [ ] 文章有3层结构（热点/深度/数据）

**权力：** 不参与任何内容生产，仅执行质检。发现问题直接驳回。

**验证证据块示例：**
```
验证证据:
- article.html: 12.4KB ✅
- SVG面板角色检查: panel-001 ✅ panel-002 ✅ panel-003 ✅ panel-004 ✅
- 数据表行数: 10 ✅
- 合规声明: 存在 ✅
- 可疑表述: 0处 ✅
- audio.mp3: 813KB ✅
```

### C-09 发布工程师
**输入：** 通过质检的全部产出
**输出：** 已发布的 GitHub Pages 网站

**流程：**
1. `python scripts/update-index.py`
2. `git add .`
3. `git commit -m "扬说财经{版次} {日期}"`
4. `git push origin main`
5. 验证: 请求网站URL确认可访问

### D-10 复盘分析师
**输入：** 当日全部产出 + 质量报告
**输出：** `docs/review/daily-{YYYY-MM-DD}.md`

**内容：**
1. 当日生产总结（产出物清单）
2. 问题记录（类型/严重度/处理方式）
3. **四原则审计**
4. 改进项（落实到下轮生产的清单）

**四原则审计格式：**
```
===== 四原则审计 =====
先思考再编码: ✅ Think假设块已覆盖[具体内容]
简单至上:     ✅ [具体证据]
精准变更:     ✅ [具体证据]
目标驱动:     ✅ 质检[分数]PASS后才部署
```

## 否决权与升级流程

| 场景 | 操作 | 升级至 |
|------|------|--------|
| Agent 2 发现>30%数据无来源 | 退回 Agent 1 + 标记来源 | 主编 |
| Agent 3 发现新闻简报质量差 | 退回 Agent 2 | 主编 |
| Agent 8 发现事实错误 | 退回 Agent 2 + 具体修改建议 | 主编终审 |
| Agent 8 发现排版/合规问题 | 退回对应生产Agent + 依据 | 主编终审 |
| Agent 9 部署失败 | 停止 + 输出错误日志 | 主编 |
