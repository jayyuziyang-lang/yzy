## 核心架构

```
四条不可绕过质量门禁：
  Layer 1: 事实核查门禁 ── 每个数据点有来源，无来源=不发表
  Layer 2: 风格一致性门禁 ── 必须匹配风格样本
  Layer 3: 质量检查门禁 ── 6故事/结构/合规/排版
  Layer 4: 数据准确性门禁 ── 可回溯零容忍

两条铁律：
  - 数据不对整篇重做（5.22编造数据的教训）
  - 标多少就做多少（5.23音频时长的教训）

风格样本库：
  - 日間模板: 央行法修订专题文章（配色/字体/间距/色值参照）
  - 每次生产前必须读样本 → 写完后逐项比对
```

## 质量门禁详解

### Layer 1: 事实核查门禁

**谁执行：** Codex 或 Claude Code（灵活）
**谁终审：** Claude Code（不可替代）

**核查标准：**
- 每个百分比数字 → 来源URL
- 每个价格数字 → 具体市场+时间
- 每个历史对比 → 精确参照年份
- 英文公司名 → 首次出现中英对照

**触发阻塞：**
- 发现编造数据 → 整篇退回，根因分析
- 关键数据无来源 → 删除该数据点
- 来源为模型"内部知识" → 不合规

### Layer 2: 风格一致性门禁

**谁执行：** Claude Code（不可替代）
**检查项：**
- 配色方案匹配风格样本
- 字体/字号/间距一致
- 6故事分类色值正确（红绿紫蓝金银橙）
- 漫画素描风格（线稿为主，色彩克制）

### Layer 3: 质量检查门禁

**谁执行：** Claude Code（不可替代）
**检查清单详见底部。**

### Layer 4: 数据准确性门禁

**谁执行：** Claude Code（不可替代）
**检查项：**
- "历史第X次""史上最大""创X年最大"等表述 → 停，必须有精确来源
- 所有数据可回溯到来源
- 无模型内部知识替代实时搜索

## 写作质量标准

### 每条故事的黄金结构

```
[数据] → 发生了什么（精确数字+时间背景）
[叙事] → 为什么会这样（逻辑推演+上下文）
[洞察] → 意味着什么（有态度的判断）
```

### 具体规范

1. **数据精确**：小数点后至少2位，来源可查
2. **有因果**：不报流水账，解释为什么涨/跌
3. **有态度**：每条故事必须有阿扬的判断，不是中立总结
4. **禁止套话**：不写"值得关注""后市可期""需警惕"
5. **禁止编造历史对比**：不写"历史第X次""创X年最大"——除非有精确来源
6. **公司名**：首次出现中英对照（如"超威半导体AMD"），后续只用中文
7. **禁止使用模型内部知识**：每个数据点必须来自实时搜索

### 数据准确性铁律

**P0级别违规——发现即阻塞发布：**
- "史上最大""历史第X次""创N年最大"等表述 → 必须验证到精确来源
- 任何百分比/价格数字无来源 → 删除
- 音频/阅读时间标注与实际不符 → 修正

## 文章风格规范

### 风格样本位置

`special/央行法修订/article.html` — 参考此文的完整CSS结构和视觉语言。

### 核心结构

```
.top-nav → 固定顶部导航条（蓝底白字返回+日期+版次标签）
.header → 蓝渐变背景（#1A56DB→#1E3A7A）:
  ├─ .series-tag — 版次标签（白字金色数字）
  ├─ h1 — 标题
  └─ .meta — 日期+标签+阅读时间
.player → 音频播放器区域（白底圆角）
  ├─ 图标+标题+描述
  └─ <audio> 标签
.body → 正文内容区（padding: 20px）
  ├─ .lead — 开篇引语（金色左边框4px + 浅灰背景）
  ├─ .section-block × N — 每故事独立区块
  │   ├─ .section-title → .section-num（有色圆形编号）+ .tag（分类标签）+ 标题文字
  │   ├─ .section-content → 段落文字 + .highlight（金黄强调）+ .key-point（蓝色重点）
  │   ├─ .section-img → 漫画SVG + 说明文字
  │   └─ table 可选 → 数据表格
  ├─ .section-divider → 金色渐变分隔线
  ├─ .pull-quote → 蓝色居中金句
  ├─ .sources → 琥珀色背景来源区
  ├─ .end-box → 蓝渐变收尾框
  └─ .warn → 红色免责声明框
```

## 漫画质量标准（核心特色部门）

### 核心理念

漫画是扬说财经的**核心特色**，但核心目的是**辅助理解**，不是炫技。每张漫画必须有明确的信息传递目标——解释难懂的逻辑、突出重要数据、让读者一眼看懂发生了什么。

**漫画数量不固定，根据需要配图。** 聚焦于关键热点和复杂逻辑，不为了凑数而画。

### 风格基准：双轨制

我们维护**两套风格基准**，分别用于日常版和专题版：

| 场景 | 参照文件 | 定位 | 核心特征 |
|------|---------|------|---------|
| **日常版**（早报/晚报） | `2026-05-22/finance_daily_20260522.html` | 快速信息传递 | 简洁线稿，几何图形，数据驱动 |
| **专题/深度版** | **`special/央行法修订/comic/panel-*.svg`** | 深度叙事理解 | **建筑学隐喻，金色+深蓝配色，叙事驱动，每图独立成篇** |

**日常版风格参照：** `2026-05-22/finance_daily_20260522.html` — 此文的SVG为本团队日常漫画的标准风格。每次生产日常内容前必须先读此参考。

**专题/深度版风格参照：** `special/央行法修订/comic/panel-*.svg` — 此系列的4张SVG为专题漫画的标杆。用户已确认"在这个基础上做的更好"。专题漫画应遵循：
- 每张图有明确的视觉隐喻（柱基/门/水流/对比等建筑学意象）
- 配色体系：暖白底 `#FAFAFA` + 深蓝 `#1E3A5F` + 金色 `#D4A017` + 灰色辅助
- 每张图独立成篇，讲述一个完整概念
- 文字排版讲究，多行对齐，信息分层
- **每次生产专题内容前必须先读此参考。**

### 底线风格规范（两组统一，不可擅自更改）

**风格参数（强制）：**

| 维度 | 标准 | 说明 |
|------|------|------|
| 背景色 | `#faf8f5`（暖白） | 不渐变，无纹理 |
| 线条色 | `#1a1a1a`（近黑） | 2-3px线宽，无阴影 |
| 金色 | `#c9a96e` | 用于强调/趋势线/关键数值 |
| 红色 | `#c62828` | 用于下跌/风险/警示 |
| 绿色 | `#2e7d32` | 用于上涨/利好 |
| viewBox | `300 220` | 固定尺寸，不缩放 |
| 字体 | `PingFang SC, Microsoft YaHei` | sans-serif |
| 人物/角色 | **不加入** | 漫画不上人物形象 |
| 装饰效果 | **无** | 无渐变、无投影、无filter、无发光 |
| 文件大小 | < 5KB | 保持简洁 |

**文字排版规则（强制）：**
- 文字必须放在空白区域，不得与图形重叠
- 重要数字用大号粗体（11-16px），说明用小号（7-8px）
- 上下左右保留充足边距

**任何风格变更（背景色、尺寸、配色、新增元素）必须先展示预览，用户确认后才能发布。**

### 漫画生产流程

```
1. 分析文章 → 确定哪些板块需要漫画（非必需不画）
2. 确定版型 → 日常版参照5.22日报 / 专题版参照央行法修订系列
3. 设计视觉隐喻 → 日常用几何图形+数据 / 专题用建筑学隐喻+金色深蓝色系
4. 编写SVG → 严格按对应风格基准
5. 质量检查 → 文字遮挡/风格匹配/数据准确/文件<5KB
```

### SVG禁止事项

- ❌ 禁止加入任何人物形象/角色/头像
- ❌ 禁止使用深色/渐变背景
- ❌ 禁止使用投影/发光等装饰滤镜
- ❌ 禁止强行加入品牌标识/水印
- ❌ 禁止文字与图形重叠
- ❌ 禁止擅自改变配色方案

## 数据可视化（Python工具链）

### 核心理念

Python图表工具（`scripts/generate-charts.py`）是我们已有的强大资产。数据图表 ≠ 漫画，两者互补：
- **漫画**：解释逻辑、视觉隐喻、让读者"看懂"发生了什么
- **数据图表**：展示真实数据走势、让读者"看清"数据的全貌

**两者在文章中同时使用，不互相替代。**

### 可用脚本

| 脚本 | 功能 | 产出 |
|------|------|------|
| `scripts/generate-charts.py` | 生成8张标准数据图表 | `docs/charts/*.svg` |
| `scripts/generate-xkcd-charts.py` | 生成手绘风格图表 | `docs/charts/` |
| `scripts/article-template.py` | 生成标准化文章HTML | 文章html |

### 图表清单（generate-charts.py产出）

| 图表 | 内容 | 适用板块 |
|------|------|---------|
| `nvidia_revenue.svg` | 英伟达单季营收柱状图 | AI/科技 |
| `nvidia_net_income.svg` | 英伟达净利润柱状图 | AI/科技 |
| `gold_price_trend.svg` | COMEX黄金期货走势 | 商品/黄金 |
| `central_bank_gold.svg` | 各国央行购金量排行 | 宏观 |
| `pboc_gold_reserves.svg` | 中国央行黄金储备趋势 | 宏观 |
| `astock_indices.svg` | A股主要指数涨跌幅 | A股 |
| `sector_flow.svg` | A股板块资金流向 | A股 |
| `oil_price.svg` | WTI原油期货走势 | 商品 |

### 使用流程

```bash
# 每天文章撰写前，先运行图表生成
python scripts/generate-charts.py

# 图表输出到 docs/charts/ 目录
# 在文章 HTML 中用 <img> 嵌入相关图表到数据板块
```

### 强制要求

- ✅ 每天至少运行一次 `generate-charts.py`（产出实时数据图表）
- ✅ 文章中嵌入数据图表的板块（如A股指数、板块资金流向等）
- ✅ 图表嵌入在文章的Layer 3 "数据一览" 或对应板块
- ❌ 不要将数据图表当作漫画使用（两者功能不同）

### 流程

1. 撰写 script.txt（阿扬人格化，第一人称）
2. 合成MP3：`python -m edge_tts --voice zh-CN-YunyangNeural --rate=-5% -f script.txt --write-media audio.mp3`

### 脚本规范

- 纯中文文本，无英文字母/URL/HTML标签
- 金融专有名词（NVIDIA/OpenAI/SpaceX等）：允许保留，但尽量用中文
- 口语化表达（"昨天""隔夜那边""有意思的是"）
- 开口："你好，我是阿扬"
- 结尾："我是阿扬，我们明天见。"

### 时长标准

| 版次 | 字数 | 预计时长 |
|------|------|---------|
| 早报 | 500-800字 | 2-3分钟 |
| 晚报 | 1500-2500字 | 6-8分钟 |
| 专题 | 1600-2700字 | 6-10分钟 |

### 铁律

**标注不撒谎：** script字数→预计时长→实际时长，三者必须一致。标8分钟就不能是5分钟。（复盘P0-02的教训）

## 质量门禁清单

### Claude Code终审检查清单

```
□ 文章 > 10KB
□ 6故事结构完整（或合理跳过）
□ 每条故事有数据+叙事+洞察
□ docs/charts/ 目录当天有更新（generate-charts.py已运行）
□ 文章中嵌入了相关数据图表
□ 所有漫画面板存在且 < 5KB（线稿SVG标准）
□ 日常漫画风格匹配5.22日报样本
□ 专题漫画风格匹配央行法修订样本（暖白底/深蓝+金色/建筑学隐喻）
□ 漫画无文字遮挡（文字在空白区域，不重叠）
□ 漫画无人物形象/角色/头像
□ 漫画无深色背景/渐变/投影等装饰
□ 无"历史第X次"类可疑表述
□ 无"史上最大"类无来源表述
□ script.txt 可用（无英文/标签问题）
□ audio.mp3 存在且>100KB
□ article.md 存在
□ 合规声明存在（"不构成投资建议"）
□ 无模板占位符残留
□ 数据索引已更新
□ 音频时长与页面标注差异≤20%
□ 脚本字数/预计时长/实际时长一致
```

**任何一条不通过 → 不发布，先修复。**
