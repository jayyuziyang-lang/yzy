## 三、整改措施

### 🛠 S1: 将散落规则集中到审核脚本（立即执行）

在 `audit-article.py` 中增加审核维度：

| 新增审核项 | 触发条件 | 检查方式 |
|-----------|---------|---------|
| gold-box风格检查 | 晚报 | gold-box内容不能以"？/?"结尾 |
| 图表数据日期 | 早报+晚报 | 检查article.html中引用的图表路径是否含日期或为本地charts/目录 |
| 字数下限检查 | 晚报 | script.txt 汉字数 < 1200 → 警告，< 800 → 阻塞 |
| 每story配图检查 | 晚报 | 6个story块是否都有 story-img |
| 图表路径检查 | 所有 | 禁止引用 `docs/charts/*.svg`（旧图表目录）|

### 🛠 S2: 修复pre-deploy-check.sh（立即执行）

pre-deploy-check.sh 项8 改为：
- 先检查哪个版本（morning/evening）有内容
- 只有对应版本才审核
- 两个版本都有则都审核
- 都没有则跳过

### 🛠 S3: 新增"生产前自检清单"硬性步骤

在 WORKFLOW.md Phase 0 中增加以下检查，不通过不进入 Phase 1：

```
□ [图表] 所有图片路径指向当日/本地目录，非 docs/charts/
□ [晚报] gold-box使用陈述句（grep检查gold-box内容）
□ [字数] script.txt 汉字数达标（早报≥500, 晚报≥1200）
□ [story] 每个story已嵌入img（grep检查story-img数量≥6）
```

### 🛠 S4: memory更新

更新 `evening_ending_style.md` 增加示例样本，确保下次不犯。
