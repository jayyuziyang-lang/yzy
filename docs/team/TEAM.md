## 核心制度变更

### 1. Think假设块（每个Agent启动时）

每个Agent开始工作前必须输出：

```
Think 假设:
1. [假设内容] — 确信度: 高/中/低
2. [假设内容] — 确信度: 高/中/低
```

目的：**暴露不确定性，让下游Agent和主编能发现潜在风险。** 如果你的假设是错的，下游能立即发现并制止。

### 2. 验证证据块（每个Agent完成时）

不能只说"做好了"，必须有客观证据：

```
验证证据:
- 已完成项: [具体清单]
- 检查结果: [PASS/FAIL/WARN]
- 发现的异常: [如有]
```

### 3. 质检审查官独立门禁

Agent 8 不参与任何内容生产，仅执行质检。发现有质量问题，直接驳回：

- 事实错误 → 退回 Agent 2 重新核查
- 排版问题 → 退回 Agent 4 重新设计
- 数据不一致 → 退回 Agent 6 重新生成
- 驳回必须附带具体理由 + 修改建议

### 4. 四原则审计（每次生产结束后）

```
===== 四原则审计 =====
先思考再编码: ✅/❌ [证据]
简单至上:     ✅/❌ [证据]
精准变更:     ✅/❌ [证据]
目标驱动:     ✅/❌ [证据]
```

## 生产流水线

### 晚报生产流程

```
Phase 0: 环境验证
  → python --version, git status, edge-tts 可用
  → 验证生产目录就绪
  → Think块输出

Phase 1: 新闻采集 (Agent 1)
  → WebSearch 搜索国内外财经新闻
  → 输出: docs/review/news-evening-{date}.md
  → 验证证据块

Phase 2: 事实核查 (Agent 2)
  → 对每个数据点追溯来源
  → 标记可疑/无法验证的数据
  → 输出: docs/review/factcheck-{date}.md
  → 未通过则阻塞流水线

Phase 3: 内容生产 (Agent 3-7 尽可能并行)
  → Agent 3: 文章撰写 (article.html + article.md)
  → Agent 4: 漫画设计 (panel-*.svg × 3-4格)
  → Agent 5: 音频制作 (script.txt + audio.mp3)
  → Agent 6: 数据可视化 (generate-charts.py)
  → Agent 7: PDF深度分析 (6篇×4维度)

Phase 4: 质量审查 (Agent 8)
  → 客观证据收集
  → 可读性/合规/数据一致性检查
  → 通过/驳回决定

Phase 5: 发布部署 (Agent 9)
  → 更新索引
  → git commit + push
  → 验证部署

Phase 6: 复盘 (Agent 10)
  → 当日问题总结
  → 四原则审计
  → 改进项记录
```
