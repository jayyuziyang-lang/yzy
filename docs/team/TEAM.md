## 14人团队编制

| 编号 | 角色 | Agent名称 | 职责 |
|------|------|-----------|------|
| **D-01** | **运营总监** | **Claude (主脑)** | **全局统筹、质量门禁、应急决策** |
| | | | |

### 新闻采集组（2人）

| 编号 | 角色 | Agent文件 | 职责 |
|------|------|-----------|------|
| A-01 | 国内财经记者 | `news-collector.sh domestic` | 每日A股/政策/行业新闻采集，每条附带来源 |
| A-02 | 国际财经记者 | `news-collector.sh international` | 每日美股/大宗/地缘新闻采集，交叉验证 |

**质量要求：**
- 每条新闻必须标注来源（URL + 媒体名称）
- 关键数据必须有2个以上独立来源交叉验证
- 禁止使用模型内部知识替代实时新闻搜索

### 内容生产组（2人）

| 编号 | 角色 | 职责 |
|------|------|------|
| B-01 | 早报主编 | 早报内容策划、storyboard编写、三层文章结构 |
| B-02 | 晚报主编 | 晚报深度分析、延长版内容、数据融入叙事 |

**质量标准：**
- 文章必须基于已验证的新闻源
- storyboard面板数与文章层级对等
- 晚报正文>=5000字，早报>=3000字

### 视觉设计组（3人）

| 编号 | 角色 | 职责 |
|------|------|------|
| C-01 | 数据可视化师 | 图表生成、数据准确验证、视觉风格统一 |
| C-02 | 漫画分镜师 | SVG漫画创作、角色表情迭代、视觉叙事 |
| C-03 | 品牌设计师 | 品牌资产维护、封面/头像迭代、视觉调性 |

**迭代机制：** 每月通过`comic-stylist.sh`研究行业趋势，更新漫画风格。

### 音频制作组（2人）

| 编号 | 角色 | 职责 |
|------|------|------|
| D-01 | 脚本撰稿人 | 口语化脚本撰写、语调节奏设计 |
| D-02 | 音频导演 | 音色选择、语速调节、音频质量把控 |

**流程：** 脚本 → 专人审核 → 音频生成（不可跳过审核步骤）

### 质量控制组（2人）

| 编号 | 角色 | Agent文件 | 职责 |
|------|------|-----------|------|
| E-01 | 内容审核员 | `fact-checker.sh` | 事实核查、数据一致性验证、源引用完整性 |
| E-02 | 技术质检员 | `quality-check.py` | 49项自动检查、文件完整性、断链检测 |

**门禁规则：** 任何模块未通过质量检查，不得进入下一生产环节。

### 运营复盘组（2人）

| 编号 | 角色 | Agent文件 | 职责 |
|------|------|-----------|------|
| F-01 | 每日复盘员 | `daily-review.sh` | 每日工作总结、问题追踪、改进日志 |
| F-02 | 竞争分析员 | `competition-analyst.sh` | 研究顶尖财经媒体、对标差距、趋势报告 |

## 每日工作流

### 早报流程 (用户: "给我今天的早报")

```
运营总监(Claude)
  │
  ├── 1. 指派 A-01/A-02 → 新闻采集
  │     └── WebSearch × N 轮 → 结构化新闻简报(含来源)
  │
  ├── 2. 指派 E-01 → 事实核查
  │     └── 验证关键数据、交叉确认
  │
  ├── 3. 指派 B-01 → 内容策划
  │     └── 基于已验证新闻 → storyboard.json + article写作
  │
  ├── 4. 触发生产管线
  │     └── bash scripts/morning.sh
  │     └── 阶段: 图表 → 品牌 → 漫画 → 脚本 → [审核] → 音频
  │
  ├── 5. 指派 E-02 → 49项质量检查
  │
  └── 6. 交付 → 等待用户审核脚本
```

### 晚报流程 (下午/晚上)

```
与早报结构一致，session=evening
```

### 每日复盘 (当天工作结束后)

```
运营总监(Claude)
  ├── 1. 指派 F-02 → 竞争分析
  │     └── 研究当天顶尖财经内容、记录可借鉴点
  │
  ├── 2. 指派 F-01 → 每日复盘
  │     └── 总结当天问题、记录改进、更新团队能力
  │
  └── 3. 更新docs/review/ → 存档
```

## 质量承诺

1. **每一条新闻数据都有来源** — 拒绝"模型已知信息"
2. **每一次交付都经过验证** — 49项质量检查通过才交付
3. **每一天都在进步** — 复盘机制确保问题不过夜
4. **每一种风格都跟得上时代** — 月度漫画风格迭代+竞品分析
