## 二、参考文章核心品质解码

### 2.1 数据密度标准

参考文章中的典型数据密度（以黄金一节为例）：

```
现货黄金 XAU $4,539.70 ▼ −0.04%
● 金价自两月低位反弹，重新站上$4,500。
   上周金价一度跌破$4,400触及两个月低点，
   本周一回升至$4,539一线，较低位反弹约1%。
● 回升动力来自两点：
   ①前期跌幅过深、超卖后的技术性修复；
   ②油价反弹+协议未签，地缘溢价与通胀担忧重新被计入。
● 核心矛盾未变：金价仍由"降息预期"而非"通胀对冲"定价。
   逻辑链：油价上行→输入性通胀担忧升温→美联储降息被推后
   →实际利率维持高位→黄金机会成本上升→金价承压
```

**关键特征提取：**
- 每个数据点附带精确价格和时间（$4,539.70，两月低位）
- 上涨/下跌拆分为多因素贡献（①和②）
- 提出核心矛盾判断（"核心矛盾未变"）
- 给出完整的5步传导链
- 反向条件判断（"一旦霍尔木兹确认重开→..."）

### 2.2 情景分析框架

参考文章将不确定性转化为可交易的框架：

```
【路径A · 协议落地】
特朗普与哈梅内伊签字 → 霍尔木兹30天内复航
→ 滞压油轮放行 → 布伦特塌回$75–$80
→ 通胀降温 → 降息预期复苏 → 美元承压
→ 黄金打开主升 · 美股再创新高

【路径B · 谈判继续拖延】
第二阶段核谈判僵持 → 协议迟迟不签
→ 空头回补、油价$90–$100高位震荡
→ 核心PCE维持3.3%+ → 沃什6月按兵不动
→ 实际利率高企 → 黄金承压 · 美股高位震荡
```

**关键特征提取：**
- 每路径5-7步完整传导链
- 每条路径在末端标注对各类资产的影响
- 路径之间互斥且覆盖全部可能
- 附有"独立主线"（不依赖停火的AI叙事）

### 2.3 时间维度管理

参考文章在同一话题内管理多个时间层：

| 时间层 | 内容 | 占比 |
|--------|------|------|
| 过去24h | 周末协议进展、周五收盘数据 | 40% |
| 今日 | 黄仁勋演讲、ISM制造业PMI | 30% |
| 本周 | 非农、ADP、COMPUTEX | 20% |
| 6月展望 | FOMC、降息路径 | 10% |

对比我们早报：几乎全部集中在"过去几天"的静态回顾，缺少前瞻性时间框架。

### 2.4 细节质感提升

参考文章通过具体细节赋予内容"触感"：

| 抽象表述 | → 具体细节 |
|---------|-----------|
| "油价可能下跌" | → "426艘油轮、34艘LPG运输船、19艘LNG船只一旦获准通航" |

## 三、根因分析：我们为什么不如参考文章？

### 3.1 信息采集环节

- **参考文章**明显基于多源专业信息整理（路透社、彭博、各大投行报告、官方声明）
- **我们早报**仅依赖通用网页搜索摘要，缺乏一手专业信源

### 3.2 分析流程环节

- **参考文章**每条分析都有明确的"因果链"：从事件→传导→资产影响，层层递进
- **我们早报**更接近"新闻摘要汇编"，分析停留在表层

### 3.3 写作调性环节

- **参考文章**采用"分析师视角"：有观点、有判断、有态度、有框架
- **我们早报**采用"播报员视角"：陈述事实为主，缺乏有穿透力的判断

### 3.4 核心差距总结

```
参考文章 = 专业金融简报（机构级）
我们早报 = 自媒体新闻摘要（个人级）

差距本质不是字数，而是四个"有没有"：
1. 有没有一手信源？→ 没有
2. 有没有因果传导链？→ 单层
3. 有没有多情景分析？→ 没有
4. 有没有明确观点判断？→ 模糊
```

## 四、升级方案

### 4.1 每日早报品质升级标准（v2.0）

从今天起，每日早报必须达到以下标准：

#### 结构标准
```
□ 行情总览表（精确到小数点后2位）
□ 头号焦点（🔥）— 最重大事件深度分析，含因果链+双情景
□ 资产深度（🥇/🛢️）— 黄金+原油或多资产联动分析
□ 美股/科技（💻）— 含关键公司数据和财报解读
□ 宏观政策（🏦）— 含关键数据解读+央行政策逻辑
□ 日历（📅）— 本周完整财经事件时间表
□ 逻辑框架（🧠）— 路径A/路径B/独立主线的决策树
```

#### 分析标准
```
□ 每个数据点附带上下文（时间/来源/对比基准）
□ 每项分析包含完整传导链（≥3步：事件→中间变量→资产→结论）
□ 不确定性事项附带双情景分析
□ 每节末端有"逻辑链"或"核心矛盾"总结
□ 有明确的观点判断（不只是"可能"、"关注"）
```

#### 信息密度标准
```
□ 每则数据附带具体数字（如$4,539.70而非$4,500+）
□ 有场景细节描述（如"426艘油轮"而非"大量油轮"）
□ 关键人物直接引语（Trump帖文原文、官员表态原文）
□ 机构观点引用（摩根士丹利、BofA、Yardeni等）
```

### 4.2 搜索流程升级

```
旧流程：
WebSearch("A股 6月1日 新闻") → 抓3-5条结果 → 写稿

新流程（强制）：
Step 1: WebSearch("[资产/事件] 最新价格 精确点位") → 获取精确数据
Step 2: WebSearch("[事件] 最新进展 细节") → 获取场景细节+直接引语
Step 3: WebSearch("[事件] 影响分析 投行 观点") → 获取专业机构判断
Step 4: WebSearch("[事件] 历史对比 类比") → 获取历史参照
→ 四步信息汇总后，才开始写稿
```

### 4.3 写作流程升级

```
旧流程：
信息搜集 → 直接撰写6故事 → 终审

新流程（强制）：
信息搜集（4轮搜索）→ 整理数据表+关键引语 → 
提炼核心逻辑链（每话题3-5步传导链）→ 
构建双情景框架（路径A/B）→ 
撰写逐节深度分析 → L3推理验证 → 终审

每个环节之间有明确的"门禁"——未完成不可跳过。
```

### 4.4 品质检测清单

发布前自检：

```
信息密度检测：
□ 每节数据点≥5个
□ 有具体场景细节（非概括性描述）
□ 有直接引语（人物原文或机构原话）

分析深度检测：
□ 传导链≥3步
□ 有"核心矛盾"判断
□ 不确定性事项有A/B路径

结构完整检测：
□ 有行情总览表
□ 有逻辑框架（🧠）
□ 有本周日历

态度检测：
□ 有明确观点（不是"可能涨也可能跌"的中立废话）
□ 有分情景的资产方向判断
□ 无"值得关注""后市可期"等空洞表述
```
