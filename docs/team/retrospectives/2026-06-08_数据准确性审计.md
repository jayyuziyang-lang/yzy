## 根因分析

### 根因1：单一搜索结果的"自信采纳"

本次制作中，所有错误数据的共同来源是：**只看了第一轮搜索结果的摘要摘要，没有做二次验证。**

- WTI $87.50 来自某个二级来源的"Oil prices fall"搜索结果，而非WTI官方结算价
- 道指50,866 来自同一搜索结果混淆了不同的数据源
- "鲍威尔讲话"是**重大低级错误**——我"知道"周四有央行讲话，下意识归给了鲍威尔（美联储），而没有核实周三CPI（美国）和周四ECB（欧洲）的时间线关系

**整改：** 关键数据（指数收盘、商品价格、人名/机构名）必须至少2个独立来源交叉验证。央行事件必须确认"哪个央行→谁讲话"。

### 根因2：文章写完"一次性部署"，缺少数据复核环节

写完文章后虽然通过了audit-article.py审核，但该审核脚本只检查：
- 有无来源URL ✓
- 有无套话 ✓
- 结构完整性 ✓

它**不检查数据本身是否准确**。数据准确性完全依赖我写文章时的原始搜索质量。

**整改：** deploy前新增人工复核步骤：逐一比对文章中的关键数字与搜索来源的一致性。

### 根因3：搜索时间线覆盖不完整

第二轮、第三轮搜索（针对具体数据的详细搜索）几乎全部都找到了正确的数字。问题在于：**我默认第一轮搜索的结果就是准的，没有再花30秒做一次针对性核实。**

具体表现：
- 第一轮搜索摘要写"WTI $87.50" → 直接用了
- 第二轮用source:finance+price搜索 → 发现WTI实际是$90.30
- 如果反过来（先搜精确数据，再写文章），这个问题就不会出现

**整改：** 写作顺序改变：**先收集精确价格数据，再写文章段落**，而不是写到某个品种时临时搜。

```
### Phase 4.5: 数据准确性复核（新增 — 2026-06-08 复盘追加）

deploy前必须完成以下交叉验证：
1. 美股三大指数收盘价/涨跌幅 — 2个独立来源确认
2. WTI/布伦特原油收盘价 — 2个独立来源确认

5. 10年美债收益率 — 确认数据正确
6. 关键人名/机构 — 确认事件对应正确的人（鲍威尔≠拉加德）
7. 文中所有搜索到的数据点 — 至少在搜索结果中看到两次
```

### 措施B：写入长期记忆

将以下规则写入长期记忆：
- **人名/机构必须确认身份**：写"某某讲话"前先确认这个人在哪个机构、什么职位
- **价格数字双源验证**：每种资产的收盘价必须有2个独立来源支持
- **先搜数据后写文**：先收集所有关键数据，再写段落，避免边写边猜
- **deploy不得跳过数据复核**：Phase 4.5新增的数据复核步骤不能省略

## 经验教训总结

| 教训 | 对应措施 |
|------|---------|
| 单一搜索来源不可靠 | 关键数据必须有2个来源确认 |
| 写文章时"边写边搜"容易出错 | 提前收集精确价格数据再写 |
| 人名错误是低级错误但后果严重 | 写"某某讲话"前确认其在哪个机构 |
| 审计脚本不检查数据准确性 | 新增人工复核步骤 |
