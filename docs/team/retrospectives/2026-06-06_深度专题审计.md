## 根因分析

### 根因1：文档三角不一致

当前有三份文档定义音频参数，但内容互相矛盾：

| 参数 | CLAUDE.md | WORKFLOW.md | AGENT_HANDBOOK.md |
|------|-----------|-------------|-------------------|
| 早报声音 | YunyangNeural | YunyangNeural | ~~XiaoxiaoNeural~~ → ✅ 已改 |
| 早报语速 | -5% | -5% | ~~-10%~~ → ✅ 已改 |
| 深度专题语速 | 无 | ~~-50%~~ → ✅ -5% | ~~-50%~~ → ✅ -5% |

**整改：** 每次改音频参数时，必须同时更新三份文档。AGENT_HANDBOOK.md 已统一为 YunyangNeural -5%。

### 根因2：违背"已有文件优先"的核心教训

CLAUDE.md 核心教训写得清清楚楚：
> **不要看文档说什么，要看已有的文件是怎么做的。**

本次制作中：
- 第2期音频（全球央行囤金）的正确参数是 YunyangNeural -5%，但第3期没有参考已有的 audio.mp3 文件设置
- 第2期脚本的结构和质量没有被当作第3期的基准

**整改：** 深度专题自检清单新增：生成前先用 ffprobe 检查上一期 audio.mp3 的编码参数和时长，确保参数一致。

### 根因3：缺少音频完整性/时长验证

pre-deploy-check.sh 只检查文件存在性和大小（>0），没有验证：
1. MP3 文件帧结构是否完整（损坏检测）
2. 实际时长与页面标注偏差 < 20%
3. 脚本汉字字数与实际时长的匹配度

**整改：** deploy.sh 中音频生成后立即执行音频完整性检查，不通过不通关。

### 根因4：专题脚本无质量基线对标

写第3期脚本时没有先读第2期脚本，导致：
- 叙事结构强度下降（第2期有"引信→定价权易主→情景推演"的清晰递进）
- 开头冲击力不足（第2期："2026年2月，一件足以写入金融史的事情发生了"）
- 口语化表达减少，数据密度增加但洞察密度下降
- 每段缺少"所以呢"的收束

**整改：** 深度专题脚本生产流程新增"先读上一期脚本"步骤，对照六条口播质量标准（叙事线索/明确观点/数字服务论证/开头3秒/下注决议/密度为王）逐条自检。

### 措施B：深度专题生产流程新增步骤

在 WORKFLOW.md 专题生产流程中新增：

```
### Phase 0.5: 质量基线对齐（新增 — 2026-06-06 复盘追加）

生成脚本前必须完成：
1. 读取上一期专题的 script.txt（质量基准）
2. 读取上一期专题的 article.html（文章结构基准）
3. 对照 AGENT_HANDBOOK.md 口播六条质量标准自检

生成音频后必须完成：
1. ffprobe 验证音频实际时长
2. 实际时长与页面标注偏差 < 20%
3. 检查音频文件帧结构完整性

生成音频前必须验证参数：
1. 查看同专题目录下已有的 audio.mp3 文件基本信息
2. 确保参数与已有文件一致
```

### 措施C：部署前新增音频完整性检测

在 pre-deploy-check.sh 专题检查部分新增：

```bash
# 音频完整性检测（新增）
if [ -f "$topic_dir/audio.mp3" ]; then
  DURATION=$(ffprobe -v error -show_entries format=duration \
    -of default=noprint_wrappers=1:nokey=1 "$topic_dir/audio.mp3" 2>/dev/null || echo 0)
  if [ "$(echo "$DURATION < 60" | bc)" = "1" ]; then
    echo -e "  ${RED}    ❌ audio.mp3 时长过短 ($DURATION 秒)${NC}"
  fi
  # 检查文件末尾是否有大量填充字节（损坏标志）
  TAIL_BYTES=$(tail -c 100 "$topic_dir/audio.mp3" | od -An -tx1 | grep -o 'aa' | wc -l)
  if [ "$TAIL_BYTES" -gt 80 ]; then
    echo -e "  ${RED}    ❌ audio.mp3 可能损坏（末尾填充字节过多）${NC}"
  fi
fi
```

### 措施D：保存到记忆

将以下规则写入长期记忆：
- 音频参数统一：所有版本（早报/晚报/深度专题）使用 YunyangNeural -5%
- 专题脚本必须读上一期脚本找质量基线
- 音频生成后必须 ffprobe 验证
- 修改文档参数时必须三份同步
