## 一、今天暴露的问题

### 问题1：名词解释"排版一致"被误解为"完全复制"（新问题 🔴）

| 维度 | 内容 |
|------|------|
| **表现** | 6.3晚报名词解释用金色主题（#D4A017表头，#FFFBEB背景），6.4早报直接复制了同样的配色。用户要求"排版一致但颜色符合早报" |
| **用户原话** | "请保持之前名词解释的排版风格" → "颜色主题风格要符合早报，蓝色" |
| **已有规则** | 早报品牌色为蓝色（#1A56DB），晚报风格色为金色（#D4A017）——在AGENT_HANDBOOK.md和brand-guidelines.md中均有定义 |
| **根因** | ① 将"排版一致"等同于"完全复制"——忽略了早报/晚报有独立的颜色主题 ② 复制6.3晚报HTML时只改了文字内容，没考虑CSS颜色属性需要适配早报 ③ 缺少"跨版本复制时逐属性检查颜色"的强制步骤 |

### 问题2：VIX标签可读性两次反馈才改到位（重复错误 🔴）

## 三、新质量门禁标准

### 门禁A：跨版本样式适配检查（新增）

每次从一个版本（晚报→早报、早报→晚报、专题→日报）复制代码时，必须逐属性检查CSS颜色是否需要适配：

```python
# 适配检查清单（复制后逐项核对）
# □ 背景色 — 符合目标版本的品牌色吗？
# □ 边框色 — 符合目标版本的品牌色吗？
# □ 标题色 — 符合目标版本的品牌色吗？
# □ 表头色 — 符合目标版本的品牌色吗？
# □ 强调色 — 符合目标版本的品牌色吗？
```

已知品牌色对照：

| 属性 | 早报（蓝色系） | 晚报（金色系） |
|------|--------------|--------------|
| 主色 | #1A56DB | #D4A017 |
| 浅背景 | #EFF6FF | #FFFBEB |
| 边框 | #BFDBFE | #FDE68A |
| 深色标题 | #1E3A7A | #92400E |

### 门禁B：用户反馈闭环标准（新增）

用户提出修改请求时的响应标准：

| 用户用语 | 最低执行标准 | 建议执行标准 |
|----------|------------|------------|
| "提升可辨识度" | 字号+粗细+颜色对比度三者都调 | 字号+粗细+颜色+加文字背景框 |
| "不够明显" | 字号+粗细同时调整 | 调完后截图/预览确认 |
| "参考XXX风格" | 列出XXX的关键视觉属性与当前版本的差异 | 逐属性适配，列出修改清单 |

**铁律：用户第一次反馈就做到位，不留到第二次。**

### 门禁C：修改后验证（新增）

涉及多个文件/输出时：

```
修改前：
  □ 确认影响范围（哪些文件？哪些输出？）

修改后：
  □ 确认输入文件已修改
  □ 确认所有输出文件已重新生成（1周图+1月图都要看）
  □ 确认修改可被用户感知
```

## 四、团队纪律

### 绝对禁止

| 禁止项 | 原因 | 替代方案 |
|--------|------|---------|
| ❌ 跨版本复制时不做颜色适配 | 早报蓝色≠晚报金色，复制即错 | 逐属性检查颜色，匹配目标版本品牌色 |
| ❌ 用户反馈只改一个维度 | 一次到位比两次修改更高效 | 字号+粗细+对比度同时调整 |
| ❌ 只改代码不验证输出 | fetch-vix-data.py改完后两个SVG都要验证 | 修改后运行脚本并检查文件 |
| ❌ 把"排版一致"当作"完全复制" | 排版=结构，颜色=风格，两者独立 | 先复制结构，再适配风格 |

### 本次铁律追加

```
铁律 #12（跨版本样式适配）：复制排版必须适配颜色
  └─ 检查清单：背景色/边框色/标题色/表头色/强调色逐属性检查
  └─ 早报 → #1A56DB蓝色系 / 晚报 → #D4A017金色系
  └─ 违反：整版退回，重新适配

铁律 #13（反馈闭环）：用户修改请求一次到位
  └─ "提升可辨识度" = 字号+粗细+颜色对比度+背景框同时调
  └─ 涉及多个输出文件时（如1周+1月图），全部验证
  └─ 违反：记过，团队复盘
```
