## 2026-06-16 晚报生产复盘

### 问题1（最严重）：Git操作不规范导致部署反复受阻

**表现：**
- deploy.sh 首次运行失败（push rejected）
- 尝试 `git pull --rebase` 被 tools/models/ 下的模型文件和lock文件阻塞
- 尝试 `git stash` 后再rebase，仍然被locked文件阻塞（Device or resource busy）
- `git rebase --abort` 失败，rebase-merge目录残留
- 急于解决问题，执行 `git reset --hard origin/main`，导致已提交的晚报文件（article.html/comic/script.txt/audio.mp3）全部丢失
- 需通过 `git checkout <commit> -- <paths>` 从 reflog 恢复文件
- 最终经历 4 次 git 操作才成功推送

**根因分析（5 Why）：**
1. 为什么push失败？→ 远程有GitHub Actions自动提交的CDN版本文件
2. 为什么pull被阻塞？→ 工作区有 tools/models/ 下的stable-diffusion模型下载残留文件（lock/incomplete/incomplete状态），与远程冲突
3. 为什么有这些残留文件？→ tools/models/ 没有在 .gitignore 中，模型下载进程在后台运行并持有文件锁
4. 为什么用 `reset --hard`？→ 急于解决问题，没有先评估风险——`reset --hard` 在 rebase 冲突中是最危险的操作
5. 根本原因：部署前无工作区检查门禁，Git操作无"危险操作确认清单"

**整改措施：**

| # | 措施 | 类型 | 立即生效 |
|---|------|------|---------|
| 1.1 | **部署前强制检查**：`git status --short` 确认无冲突文件 | 流程门禁 | ✅ 下次deploy前 |
| 1.2 | **`git reset --hard` 列入禁止操作清单**：除非用户明确要求，必须使用 `git checkout <commit> -- <paths>` 恢复特定文件，而非全量reset | 铁律 | ✅ 立即生效 |
| 1.3 | **tools/models/ 加入 .gitignore**：防止模型文件污染工作区 | 配置修复 | ✅ 本次已执行 |
| 1.4 | **rebase失败后强制清理**：`rm -rf .git/rebase-merge` + `git rebase --abort` 组合命令 | 操作规范 | ✅ 立即生效 |
| 1.5 | **推送被拒的标准操作流程**：① stash ② pull --rebase ③ stash pop ④ push。如果stash失败，先检查并清理locked文件 | SOP | ✅ 立即生效 |

### 问题3：工具链脆弱——Grep/Glob不可用，中文路径编码问题

**表现：**
- `Glob` tool 报错：`ENOENT: ripgrep binary not found at .../rg.exe`
- `Grep` tool 同样报错
- Bash 中直接使用中文路径经常编码失败

**根因：** ripgrep二进制文件路径变更或安装不完整。中文路径在Git Bash和WSL之间的编码转换不稳定。

**整改措施：**

| # | 措施 | 类型 |
|---|------|------|
| 3.1 | 搜索操作统一使用 `Bash` + `find`/`grep` 替代 `Glob`/`Grep`，直到 ripgrep 修复 | 操作规范 |
| 3.2 | 所有路径使用 `/d/Desktop/每日财经/` 前缀而非 `D:\Desktop\...` | 操作规范 |
| 3.3 | 涉及中文路径的命令优先使用 `cd` 到目标目录后再操作，减少路径转义 | 操作规范 |

### 问题5：comic SVG中VIX数据不精确

**表现：** 漫画右下角的VIX卡片写的是"持续走低"，缺少具体数值。文章对应位置也已修正为 16.20 (-8.37%)。

**根因：** 漫画与文章的数据来源不一致。漫画是手写SVG（制作时为概览版），文章是第一版写完后再补精确数据的——两者之间缺少同步机制。

**整改措施：**
- 漫画中的数据标注必须与 article.html 中的数据表完全一致
- deploy前检查清单新增：`□ 漫画SVG中的数字与article.html数据表一致`
